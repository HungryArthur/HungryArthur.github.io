В Go ошибки часто создаются как константные переменные, чтобы их можно было использовать в разных местах кода и сравнивать с помощью `errors.Is`. Это позволяет определять конкретные типы ошибок и реагировать на них соответственно. 

#### **Создание константных ошибок**
Константные ошибки создаются как глобальные переменные типа `error`, инициализированные с помощью `errors.New`. Это позволяет использовать одну и ту же ошибку в разных функциях и сравнивать её с возвращаемыми ошибками. Например:

```csharp
var ErrEmptyString = errors.New("пустая строка")
```

Такие ошибки удобны для проверки с `errors.Is`, так как они имеют уникальный адрес и могут быть сравнены напрямую.

##### **Возврат константных ошибок из функций**
Функции могут возвращать константные ошибки при определённых условиях. Это упрощает код, так как ошибки предопределены и можно легко понять, какая именно ошибка произошла. Например, если функция проверяет строку на пустоту, она может вернуть `ErrEmptyString`.

#### **Пример 1: Создание и возврат константной ошибки**
```go
package main

import (
	"errors"
	"fmt"
)

var ErrNegative = errors.New("отрицательное число")

func checkNumber(n int) (int, error) {
	if n < 0 {
		return 0, ErrNegative
	}
	return n, nil
}

func main() {
	result, err := checkNumber(-5)
	if err != nil {
		fmt.Println("Ошибка:", err)
		return
	}
	fmt.Println("Результат:", result)
}
```
**Вывод:**
```makefile
Ошибка: отрицательное число
```

**Пояснение:** Константная ошибка `ErrNegative` возвращается при отрицательном n. В `main` ошибка проверяется и выводится.

#### **Проверка ошибок с errors.Is**
Как мы уже знаем, функция `errors.Is(err, target)` проверяет, совпадает ли `err` с конкретной константной ошибкой `target`. Это полезно для реагирования на конкретные, заранее заданные константные ошибки, такие как `ErrEmptyString`. Если ошибка совпадает, `errors.Is` возвращает `true`.

#### **Преимущества константных ошибок**
- Упрощают сравнение ошибок с `errors.Is`.
- Делают код читаемым, так как ошибки имеют понятные имена.
- Избегайте повторного создания одной и той же ошибки.

Недостатки: Константные ошибки не могут содержать динамические данные (для этого используйте `fmt.Errorf`).

#### **Пример 2: Проверка с errors.Is**
```go
package main
import (
    "errors"
    "fmt"
)

var ErrNegative = errors.New("отрицательное число")

func checkNumber(n int) (int, error) {
    if n < 0 {
        return 0, ErrNegative
    }
    return n, nil
}

func main() {
    result, err := checkNumber(-5)
    if errors.Is(err, ErrNegative) {
        fmt.Println("Специфическая ошибка: отрицательное число")
        return
    }
    fmt.Println("Результат:", result)
}
```
**Вывод:**
```1c
Специфическая ошибка: отрицательное число
```

**Пояснение:** Функция возвращает `ErrNegative` при отрицательном n. В `main` `errors.Is` проверяет совпадение с константной ошибкой и выводит специальное сообщение.

#### **Пример 3: Несколько константных ошибок**
```go
package main
import (
    "errors"
    "fmt"
)

var ErrZero = errors.New("нулевое значение")
var ErrNegative = errors.New("отрицательное число")

func checkValue(n int) (int, error) {
    if n == 0 {
        return 0, ErrZero
    }
    if n < 0 {
        return 0, ErrNegative
    }
    return n, nil
}

func main() {
    result, err := checkValue(0)
    if err != nil {
        if errors.Is(err, ErrZero) {
            fmt.Println("Ошибка: нулевое значение")
        } else if errors.Is(err, ErrNegative) {
            fmt.Println("Ошибка: отрицательное число")
        } else {
            fmt.Println("Неизвестная ошибка:", err)
        }
        return
    }
    fmt.Println("Значение:", result)
}
```
**Вывод:**
```makefile
Ошибка: нулевое значение
```

**Пояснение:** Функция `checkValue` возвращает разные константные ошибки для n=0 и n<0. В `main` `errors.Is` проверяет тип ошибки и выводит соответствующее сообщение.
