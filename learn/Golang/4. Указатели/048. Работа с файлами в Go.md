Работа с файлами в Go позволяет программам считывать данные из внешних источников и сохранять результаты для последующего использования. Пакет `os` предоставляет инструменты для открытия, чтения, записи и управления файлами. Важно проверять ошибки операций с файлами и закрывать файлы, открытые с помощью `os.Open` или `os.Create`, используя `defer file.Close()` для предотвращения утечек ресурсов. В этом шаге мы рассмотрим пример чтения строки из файла, её обработки с помощью метода структуры и записи результата в другой файл, используя прямой ввод/вывод без буферизации.

**Пример:**
```go
package main

import (
	"errors"
	"fmt"
	"os"
	"strings"
)

func main() {
	// Проверяем существование входного файла
	_, err := os.Stat("input.txt")
	if err != nil {
		fmt.Printf("Error: ошибка при открытии файла input.txt: %v\n", err)
		return
	}

	// Открываем файл для чтения
	inputFile, err := os.Open("input.txt")
	if err != nil {
		fmt.Printf("Error: не удалось открыть файл: %v\n", err)
		return
	}
	defer inputFile.Close()

	// Читаем содержимое файла
	data := make([]byte, 1024) // Буфер для чтения
	count, err := inputFile.Read(data)
	if err != nil {
		fmt.Printf("Error: не удалось прочитать файл: %v\n", err)
		return
	}

	// Создаём или открываем файл для записи
	outputFile, err := os.Create("output.txt")
	if err != nil {
		fmt.Printf("Error: не удалось создать файл: %v\n", err)
		return
	}
	defer outputFile.Close()

	// Записываем данные в файл
	_, err = outputFile.WriteString(strings.ToUpper(string(data[:count])))
	if err != nil {
		fmt.Printf("Error: не удалось записать файл: %v\n", err)
		return
	}

	fmt.Println("Success")
}
```
**Вывод:**
```nginx
Success
```

**Пояснение:** Программа открывает файл `input.txt` с помощью `os.Open`, проверяя его существование через `os.Stat`. Файл считывается в буфер с помощью `inputFile.Read` . Далее содержимое преобразуется в строку и преобразует содержимое в верхний регистр. Результат записывается в `output.txt` с помощью `os.Create` и `outputFile.WriteString`. Файлы закрываются с помощью `defer`. Если `input.txt` содержит "Hello, Go!", то `output.txt` будет содержать "HELLO, GO!". Все ошибки обрабатываются с выводом сообщений, а успешное выполнение подтверждается сообщением "Success".

**Пример содержимого файла input.txt:**
```undefined
Hello, Go!
```

**Пример содержимого файла output.txt после выполнения:**
```undefined
HELLO, GO!
```

### Права доступа к файлам в Go

В Go права доступа задаются при создании файла (например, через `os.Create` или `os.WriteFile`) в восьмеричной системе, например, `0644`. Права определяют, кто может читать (`r=4`), писать (`w=2`) или выполнять (`x=1`) файл: владелец, группа или другие. Например, `0644` даёт владельцу чтение и запись (6=4+2), а группе и другим — только чтение (4).

**Комментарий:** Используйте `0644` для текстовых файлов и `0755` для исполняемых. Проверяйте права через `os.Stat` и `FileInfo.Mode()`.

|Код|Права владельца|Права группы|Права других|Описание|
|---|---|---|---|---|
|0644|rw- (6)|r-- (4)|r-- (4)|Владелец: чтение/запись; остальные: чтение|
|0600|rw- (6)|--- (0)|--- (0)|Только владелец: чтение/запись|
|0755|rwx (7)|r-x (5)|r-x (5)|Владелец: всё; остальные: чтение/выполнение|
|0666|rw- (6)|rw- (6)|rw- (6)|Все: чтение/запись|
|0700|rwx (7)|--- (0)|--- (0)|Только владелец: всё|

### Использование defer для закрытия файлов и обработка ошибок

Как мы уже знаем, `defer` в Go откладывает выполнение действия до завершения функции, что идеально для закрытия файлов, открытых через `os.Open`. Это освобождает ресурсы, даже при ошибках. В этом шаге мы разберём, как использовать `defer` для закрытия файлов, как работает LIFO-порядок. В Go `defer` упрощает управление ресурсами, как в заметках или логах. LIFO-порядок похож на закрытие окон в приложении: последнее открытое закрывается первым.

**Пример 1: Считывание числа из файла с обработкой ошибок**

Программа открывает `input.txt`, считывает одно число, выводит его на консоль, и использует `defer` для закрытия файла. 
```go
package main

import (
    "fmt"
    "os"
)

// main выполняет основную логику программы
func main() {
    // Открываем файл input.txt для чтения
    inputFile, err := os.Open("input.txt")
    // Подробнее про обработку ошибок будет рассказано в шаге 2.1 
    if err != nil {
        fmt.Println("Ошибка во время открытия файла!")
        return
    }
    
    // Откладываем закрытие файла
    defer inputFile.Close()

    // При завершении программы будет выполнены все отложенные вызовы, объявленные с defer
}
```

#### Пример 2: Два defer для вывода сообщений
Программа использует два `defer` для вывода сообщений, демонстрируя LIFO-порядок.
```go
package main

import "fmt"

// main выполняет основную логику программы
func main() {
    // Откладываем вывод двух сообщений
    defer fmt.Println("End")
    defer fmt.Println("Start")
}
```
**Вывод на консоль:**
```sql
Start
End
```

**Комментарий:** Два `defer` добавляют сообщения в стек. При завершении функции они выполняются в обратном порядке: сначала `"Start"`, затем `"End"`.

**Рекомендации:**
- Используйте `os.ReadFile` и `os.WriteFile` для простых операций с небольшими файлами.
- Проверяйте ошибки операций с файлами, чтобы избежать работы с несуществующими или недоступными файлами.
- Закрывайте файлы, открытые с `os.Open`, с помощью `defer file.Close()`.
- Используйте `if err != nil` для проверки ошибок открытия файла перед `defer`.
