Паника (panic) в Go — это механизм обработки критических ошибок, когда программа сталкивается с ситуацией, из которой не может продолжить нормальное выполнение самостоятельно.

При вызове паники программа немедленно останавливается, выводит сообщение об ошибке и стек вызовов, аналогичный тому, что предоставляет функция `debug.PrintStack` из пакета `runtime/debug`, которую мы изучали ранее. Стек вызовов показывает цепочку функций, приведших к ошибке, что упрощает отладку. Паника похожа на исключения в других языках, но в Go она предназначена для серьёзных ошибок, таких как, например, деление на ноль или некорректные аргументы в функциях стандартной библиотеки.

Паника может возникать автоматически (например, при передаче некорректного значения в функцию стандартной библиотеки) или вызываться программистом с помощью функции `panic`, которая принимает сообщение об ошибке или значение любого типа. После вызова `panic` выполнение текущей функции прекращается, и Go начинает раскручивать стек вызовов, выполняя все отложенные функции `defer`.

**Пример 1: Автоматическая паника при обращении к несуществующему индексу строки**

```go
package main

import "fmt"

func main() {
    str := "hello"
    
    // Попытка обратиться к несуществующему индексу строки
    fmt.Println("Длина строки:", len(str))
    fmt.Println("Пытаемся получить символ с индексом 10...")
    
    // Это вызовет панику: runtime error: index out of range [10] with length 5
    char := str[10] // ПАНИКА!
    fmt.Println("Символ:", char)
}
```
**Вывод:**
```go
Длина строки: 5
Пытаемся получить символ с индексом 10...
panic: runtime error: index out of range [10] with length 5

goroutine 1 [running]:
main.main()
	/tmp/sandbox3384684667/prog.go:13 +0xbd
```

**Пояснение:** Передача индекса 10 в строку длиной 5 символов вызывает автоматическую панику с детализированным сообщением об ошибке, указывающим проблемный индекс и фактическую длину строки., так как это попытка доступа к несуществующему индексу строки.

 **Пример 2: Автоматическая паника при делении на 0**
```go
package main

import "fmt"

func main() {
	a := 10
	b := 0

	// Деление на ноль вызовет панику
	result := a / b // panic: runtime error: integer divide by zero
	fmt.Println("Результат:", result)
}
```
**Вывод:**
```go
panic: runtime error: integer divide by zero

goroutine 1 [running]:
main.main()
	/tmp/sandbox4129265952/prog.go:10 +0x9
```

**Пояснение:** Деление целого числа на ноль вызывает панику, так как эта математическая операция не имеет определения. Go прерывает выполнение с сообщением об ошибке.

#### **Когда использовать панику самостоятельно**

Паника предназначена для случаев, когда ошибка настолько серьёзна, что продолжение работы программы невозможно или небезопасно. Примеры включают:
- Некорректная конфигурация программы (например, отсутствие инициализации критически важной переменной).
- Ошибки в логике, указывающие на дефект в коде (например, недопустимое состояние).
- Автоматические ошибки рантайма, такие как передача некорректного значения в функцию стандартной библиотеки.

Для обычных ошибок, таких как неверный ввод пользователя, используйте ошибки.

Синтаксис самостоятельного вызова паники прост – `panic(value)`, где `value` — это значение, которое может быть любого типа, часто — строка с сообщением об ошибке. 

Например:
```go
// Синтаксис вызова функции panic 
panic("сообщение об ошибке") 

// Или с любой другой существующий тип value 
panic(42) 
panic(someVariable) 
panic(fmt.Errorf("ошибка: %v", err)) 
```

Разберем несколько примеров самостоятельного вызова паники.

**Пример 3: Паника при делении на ноль**
```go
package main

import "fmt"

func divide(a, b int) int {
	if b == 0 {
		panic("Деление на ноль")
	}
	return a / b
}

func main() {
	result := divide(10, 0)
	fmt.Println(result)
}
```
**Вывод:**
```go
panic: Деление на ноль

goroutine 1 [running]:
main.divide(...)
	/tmp/sandbox3898810946/prog.go:7
main.main()
	/tmp/sandbox3898810946/prog.go:13 +0x25
```

**Пояснение:** В отличие от примера 2, в этой программе функция `divide` вызывает `panic` самостоятельно при b=0. Программа останавливается, выводится сообщение и стек вызовов, аналогичный `debug.PrintStack`, указывающий на строку main.go:6.

**Пример 4: Паника при некорректном значении**
```go
package main

import "fmt"

func checkPositive(x int) {
	if x < 0 {
		panic("Отрицательное значение недопустимо")
	}
	fmt.Printf("Значение: %d\n", x)
}

func main() {
	checkPositive(-5)
}
```
**Вывод:**
```go
panic: Отрицательное значение недопустимо

goroutine 1 [running]:
main.checkPositive(0xc000002380?)
	/tmp/sandbox2658187332/prog.go:7 +0x75
main.main()
	/tmp/sandbox2658187332/prog.go:13 +0x1a
```

**Пояснение:** Функция `checkPositive` самостоятельно вызывает `panic` при x < 0. Программа завершается, выводя стек вызовов, аналогичный `debug.PrintStack`.

**Преимущества паники:**
- Быстро останавливает программу при критической ошибке.
- Предоставляет стек вызовов, аналогичный `debug.PrintStack`, для отладки.
- Упрощает обработку фатальных ошибок.

**Недостатки:**
- Нарушает нормальный поток программы.
- Чрезмерное использование усложняет код и отладку.
