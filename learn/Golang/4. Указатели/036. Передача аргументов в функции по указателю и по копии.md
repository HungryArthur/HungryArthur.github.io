В предыдущих шагах вы познакомились с понятием указателей. Но на самом деле, не осознавая этого, мы уже использовали их, например, при использовании `fmt.Scan` мы передаём адрес переменной с помощью оператора `&`, чтобы функция могла записать введённое значение в эту переменную. Например:

```go
package main
import "fmt"

func main() {
    var x int
    fmt.Scan(&x) // Передаём адрес x
    fmt.Println(x)
}
```

В этом примере `&x` позволяет `fmt.Scan` изменить значение переменной `x`.

До этого все аргументы, которые мы передавали в функции, копировались, то есть передавались **по значению**.

Теперь же подробнее разберём, как в Go осуществляется передача аргументов в функции по копии (по значению) и по указателю, и как это влияет на поведение программы.

#### Передача по копии (по значению)

В Go по умолчанию все аргументы передаются в функции **по значению**. Это означает, что функция получает копию значения аргумента, а не саму переменную. Когда вы передаёте переменную в функцию, Go создаёт её копию, и функция работает с этой копией. Изменения, сделанные в функции, не влияют на оригинальную переменную, так как копия существует отдельно.

Например, если мы передавали, число или строку в функцию, эти данные копировались, и любые изменения внутри функции не отражались на переменных вне функции. Это делает код безопасным, но ограничивает возможность изменять исходные данные.

**Пример 1:**

```go
package main

import "fmt"

func increment(n int) {
	n++ // Изменяем копию
	fmt.Printf("Внутри функции: %d\n", n)
}

func main() {
	x := 10
	increment(x)
	fmt.Printf("После функции: %d\n", x)
}
```

**Вывод:**

```undefined
Внутри функции: 11
После функции: 10
```

**Пояснение:** Переменная `x` копируется в параметр `n`. Изменение `n` не влияет на `x`, так как функция работает с копией.

#### Передача по указателю

Чтобы функция могла изменить оригинальную переменную, в Go используется передача **по указателю**. Указатель — это адрес памяти, и, передавая указатель в функцию, мы даём ей доступ к оригинальным данным. В Go даже указатель передаётся по значению (копируется адрес), но поскольку этот адрес указывает на исходную переменную, функция может изменить данные по этому адресу через **разыменование** (оператор `*`).

Мы уже видели это с `fmt.Scan`, где передача `&x` позволяет функции записать введённое значение в переменную `x`. Передача указателя полезна, когда нужно изменить данные или избежать копирования больших структур.

**Пример 2:**
```go
package main

import "fmt"

func incrementByPtr(n *int) {
	*n++ // Изменяем значение по адресу
	fmt.Printf("Внутри функции: %d\n", *n)
}

func main() {
	x := 10
	incrementByPtr(&x)
	fmt.Printf("После функции: %d\n", x)
}
```
**Вывод:**
```undefined
Внутри функции: 11
После функции: 11
```

**Пояснение:** Передаём адрес `&x`. Функция разыменовывает `*n`, изменяя оригинальное значение `x`.

**Пример 3:**
```go
package main

import "fmt"

func addPrefix(s *string) {
	*s = "Mr. " + *s // Изменяем строку
}

func main() {
	name := "Alice"
	addPrefix(&name)
	fmt.Printf("После функции: %s\n", name)
}
```
**Вывод:**
```undefined
После функции: Mr. Alice
```

**Пояснение:** Указатель `*string` позволяет функции изменить строку `name`, добавив префикс.

#### Преимущества и недостатки

Передача по копии:
- **Преимущества**: Безопасность (оригинальные данные не изменяются), простота (предсказуемое поведение).
- **Недостатки**: Копирование больших данных может быть медленным, невозможно изменить оригинал.

Передача по указателю:
- **Преимущества**: Можно изменить оригинальные данные, эффективно для больших данных (передаётся только адрес).
- **Недостатки**: Усложняет код, риск ошибок (например, разыменование `nil`).

Таким образом, общий совет по выбору типа передачи переменной следующий - используйте передачу по копии для небольших данных или когда изменения не нужны. Используйте указатели, если:
- Функция должна изменить оригинальную переменную.
- Нужно передать большие данные без копирования.
- Хотите явно показать, что функция модифицирует данные.
