Для глубокого понимания механизма рекурсии крайне полезно сначала разобраться с принципом выполнения вложенных функций, который наглядно демонстрируется с помощью **стека вызовов** (Call Stack).

**Стек вызовов** — это структура данных, которая отслеживает порядок выполнения функций в программе. Когда одна функция вызывает другую, информация о текущей функции (её адрес возврата, параметры, локальные переменные) сохраняется в стеке. После завершения вызываемой функции выполнение возвращается к предыдущей функции, а её данные удаляются из стека. Это работает по принципу "последним вошёл — первым вышел" (LIFO, Last In, First Out).

Каждый вызов функции добавляет в стек новую запись, называемую фреймом стека. Фрейм содержит:
- Адрес возврата (куда вернуться после завершения функции).
- Аргументы функции.
- Локальные переменные.

Стек вызовов помогает понять, как функции взаимодействуют и где произошла ошибка, если программа завершилась некорректно.

#### Функция debug.PrintStack
Функция `debug.PrintStack` из пакета `runtime/debug` в Go используется для дебага и позволяет выводить текущий стек вызовов в момент её вызова. Это полезно для отладки, когда нужно узнать, какие функции вызывались и в каком порядке, особенно при ошибках, таких как неверные входные данные или деление на ноль. Вывод включает имена функций, файлы и строки кода, что помогает найти проблемное место.

#### Как вызываются функции

Когда программа вызывает функцию, Go выполняет следующие шаги:
1. Создаётся новый фрейм стека для вызываемой функции.
2. Аргументы копируются в фрейм (в Go передача параметров — по значению, в случаях указателей по значению передается адрес).
3. Управление передаётся функции, а адрес возврата сохраняется.
4. После завершения функции её фрейм удаляется из стека, и управление возвращается к вызывающей функции.

Если функция вызывает другую функцию, процесс повторяется, добавляя новый фрейм в стек. `debug.PrintStack` показывает все активные фреймы, начиная с текущей функции и до `main`.

#### Применение
Стек вызовов полезен для понимания последовательности выполнения программы и отладки. Например, если программа работает некорректно из-за ошибки в глубоко вложенной функции, `debug.PrintStack` покажет весь путь вызовов, помогая определить, где возникла проблема. 

**Пример 1: Простой вызов функции с отладкой**
```go
package main

import (
	"fmt"
	"runtime/debug"
)

func process(x int) {
	if x < 0 {
		fmt.Println("Ошибка: отрицательное число")
		debug.PrintStack()
	} else {
		fmt.Printf("Обработано: %d\n", x)
	}
}

func main() {
	process(5)
	process(-3)
}
```
**Вывод** (значение адресов/имен файлов может отличаться):
```go
Обработано: 5
Ошибка: отрицательное число
goroutine 1 [running]:
runtime/debug.Stack()
	/usr/local/go-faketime/src/runtime/debug/stack.go:26 +0x5e
runtime/debug.PrintStack()
	/usr/local/go-faketime/src/runtime/debug/stack.go:18 +0x13
main.process(0xc000002380?)
	/tmp/sandbox2714253348/prog.go:11 +0x54
main.main()
	/tmp/sandbox2714253348/prog.go:19 +0x25
```

**Пояснение:** Функция `process` вызывается дважды. При отрицательном `x` она выводит стек вызовов, показывая, что `process` была вызвана из `main`. Данный вывод представляет собой стектрейс (трассировку стека вызовов) программы на Go – печать текущего состояния стека вызовов функций (от `main.main()` до `runtime/debug.Stack()`), что позволяет отследить последовательность выполнения кода в момент возникновения проблемы.

**Цифры в строках кода** (например, `:26`, `:18`, `:11`, `:19`) — это номера строк в исходных файлах, где произошёл вызов функции или сбой. **Шестнадцатеричные адреса** (например, `0x5e`, `0x13`, `0x54`, `0x25`) — это смещения (оффсеты) в машинном коде внутри скомпилированных функций, показывающие конкретное место инструкции. Адрес **`0xc000002380?`** — это указатель на память (аргумент, переданный в функцию `main.process`).

**Пример 2: Вложенные вызовы функций**

```go
package main

import (
	"fmt"
	"runtime/debug"
)

func divide(a, b int) int {
	if b == 0 {
		fmt.Println("Ошибка: деление на ноль")
		debug.PrintStack()
		return 0
	}
	return a / b
}

func calculate(x, y int) int {
	return divide(x, y) + 1
}

func main() {
	result := calculate(10, 2)
	fmt.Printf("Результат: %d\n", result)
	result = calculate(10, 0)
	fmt.Printf("Результат: %d\n", result)
}
```
**Вывод:**
```go
Результат: 6
Ошибка: деление на ноль
goroutine 1 [running]:
runtime/debug.Stack()
	/usr/local/go-faketime/src/runtime/debug/stack.go:26 +0x5e
runtime/debug.PrintStack()
	/usr/local/go-faketime/src/runtime/debug/stack.go:18 +0x13
main.divide(0x4e5398?, 0xc000110018?)
	/tmp/sandbox2723655632/prog.go:11 +0x50
main.calculate(...)
	/tmp/sandbox2723655632/prog.go:18
main.main()
	/tmp/sandbox2723655632/prog.go:24 +0x7e
Результат: 1
```

**Пояснение:** Функция `calculate` вызывает `divide`. При делении на ноль `divide` выводит стек, показывая цепочку вызовов: `main` → `calculate` → `divide`.

**Пример 3: Множественные вложенные вызовы**
```go
package main
import (
    "fmt"
    "runtime/debug"
)

func step3(z int) int {
    if z < 0 {
        fmt.Println("Ошибка: отрицательное значение")
        debug.PrintStack()
        return 0
    }
    return z * 2
}

func step2(y int) int {
    return step3(y) + 5
}

func step1(x int) int {
    return step2(x) + 10
}

func main() {
    result := step1(3)
    fmt.Printf("Результат: %d\n", result)
    result = step1(-2)
    fmt.Printf("Результат: %d\n", result)
}
```
**Вывод:**
```go
Результат: 21
Ошибка: отрицательное значение
goroutine 1 [running]:
runtime/debug.Stack()
	/usr/local/go-faketime/src/runtime/debug/stack.go:26 +0x5e
runtime/debug.PrintStack()
	/usr/local/go-faketime/src/runtime/debug/stack.go:18 +0x13
main.step3(0x4e53c8?)
	/tmp/sandbox35847209/prog.go:11 +0x50
main.step2(...)
	/tmp/sandbox35847209/prog.go:18
main.step1(...)
	/tmp/sandbox35847209/prog.go:22
main.main()
	/tmp/sandbox35847209/prog.go:28 +0x77
Результат: 15
```

**Пояснение:** Функция `step1` вызывает `step2`, а та — `step3`. При отрицательном значении `step3` выводит стек, показывая цепочку: `main` → `step1` → `step2` → `step3`.
