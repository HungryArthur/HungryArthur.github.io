#### **Введение**

В Go для работы с ошибками существуют различные функции из пакета `errors` и `fmt`, такие как, например, `errors.Is`, `errors.As` и `fmt.Errorf`. Эти функции помогают создавать и анализировать ошибки, делая обработку более точной и гибкой. В этом шаге мы разберём, как использовать эти функции для создания ошибок и анализа их причин.

#### **Создание ошибок с fmt.Errorf**

Функция `fmt.Errorf` из пакета `fmt` создаёт ошибку с форматированным текстом, включая значения переменных. Она полезна, когда нужно добавить контекст к ошибке, например, указать неверное значение или параметр.

Пример использования:

```css
err := fmt.Errorf("неверное число: %d", n)
```

Это создаёт ошибку с текстом, зависящим от значения `n`.

#### **Пример 1: Использование fmt.Errorf**
```go
package main

import (
	"fmt"
)

func checkValue(n int) (string, error) {
	if n < 0 {
		return "", fmt.Errorf("отрицательное число: %d", n)
	}
	return "Число валидно", nil
}

func main() {
	message, err := checkValue(-10)
	if err != nil {
		fmt.Println("Ошибка:", err)
		return
	}
	fmt.Println(message)
}
```
**Вывод:**
```makefile
Ошибка: отрицательное число: -10
```

**Пояснение:** Функция `checkValue` использует `fmt.Errorf` для создания ошибки с описанием отрицательного числа. В `main` ошибка проверяется и выводится.

#### **Проверка ошибок с errors.Is**

Функция `errors.Is(err, target)` проверяет, совпадает ли ошибка `err` с конкретной ошибкой `target`. Это полезно для сравнения с известными ошибками, определёнными в стандартной библиотеке или коде, например, `io.EOF`. Функция возвращает `true`, если ошибки совпадают.

Пример: Проверка, является ли ошибка `io.EOF`.

#####  **Пример 2: Проверка с errors.Is**
```go
package main

import (
	"errors"
	"fmt"
	"io"
)

func readData(s string) (string, error) {
	if s == "" {
		return "", io.EOF
	}
	return s, nil
}

func main() {
	result, err := readData("")
	if err != nil {
		if errors.Is(err, io.EOF) {
			fmt.Println("Ошибка: конец ввода")
			return
		}
		fmt.Println("Ошибка:", err)
		return
	}
	fmt.Println("Данные:", result)
}
```
**Вывод:**
```makefile
Ошибка: конец ввода
```

**Пояснение:** Функция `readData` возвращает `io.EOF` при пустой строке. В `main` `errors.Is` проверяет, является ли ошибка `io.EOF`, и выводит соответствующее сообщение.

#### **Извлечение ошибок с errors.As**
Функция `errors.As(err, target)` проверяет, можно ли преобразовать ошибку `err` в определённый тип, и если да, сохраняет её в `target`. Это полезно для анализа ошибок, созданных стандартными функциями, чтобы извлечь дополнительные данные.

Пример: Извлечение ошибки типа `*strconv.NumError` из `strconv.Atoi`.

#### **Пример 3: Извлечение ошибки с errors.As**
```go
package main

import (
	"errors"
	"fmt"
	"strconv"
)

func parseNumber(s string) (int, error) {
	n, err := strconv.Atoi(s)
	if err != nil {
		return 0, err
	}
	return n, nil
}

func main() {
	number, err := parseNumber("abc")
	if err != nil {
		var numErr *strconv.NumError
		if errors.As(err, &numErr) {
			fmt.Printf("Ошибка парсинга: %s (строка: %s)\n", numErr.Err, numErr.Num)
			return
		}
		fmt.Println("Ошибка:", err)
		return
	}
	fmt.Println("Число:", number)
}
```
**Вывод:**
```java
Ошибка парсинга: invalid syntax (строка: abc)
```

**Пояснение:** Функция `parseNumber` возвращает ошибку от `strconv.Atoi`. В `main` `errors.As` извлекает `*strconv.NumError`, чтобы получить детали ошибки, такие как проблемная строка.

#### **Когда использовать errors.Is, errors.As и fmt.Errorf**
- `fmt.Errorf`: Используйте для создания ошибок с динамическим текстом, включающим контекст (например, значения переменных).
- `errors.Is`: Используйте для проверки, совпадает ли ошибка с конкретной известной ошибкой, например, `io.EOF`.
- `errors.As`: Используйте для извлечения данных из ошибки определённого типа, например, деталей из `strconv.NumError`.

Рекомендации:
- Используйте `errors.Is` для простых проверок известных ошибок.
- Применяйте `errors.As`, если нужны дополнительные данные об ошибке.
- Создавайте понятные сообщения с `fmt.Errorf` для упрощения отладки.
