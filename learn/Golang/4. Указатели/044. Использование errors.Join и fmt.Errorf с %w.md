В Go обработка ошибок часто требует объединения нескольких ошибок или добавления контекста к существующим ошибкам. Функция `errors.Join` позволяет объединить несколько ошибок в одну, а `fmt.Errorf` с использованием спецификатора `%w` оборачивает существующую ошибку, добавляя к ней дополнительный контекст. Эти инструменты упрощают работу с ошибками, позволяя собирать и передавать информацию о множественных проблемах или уточнять причину ошибки.

#### **Функция errors.Join**
Функция `errors.Join(errs ...error) error` (доступна с Go 1.20) объединяет несколько ошибок в одну. Она принимает произвольное количество аргументов типа `error` и возвращает новую ошибку, содержащую сообщения всех непустых (`non-nil`) ошибок, разделённые переносами строк. Если все переданные ошибки равны `nil`, `errors.Join` возвращает `nil`. Это полезно для функций, которые выполняют несколько проверок, и вы хотите сообщить о всех проблемах сразу.

**Пример:**
```css
err := errors.Join(errors.New("ошибка 1"), errors.New("ошибка 2"))
```

В результате `err` содержит оба сообщения, разделённые переносом строки.

#### **Пример 1: Использование errors.Join**
```go
package main

import (
	"errors"
	"fmt"
)

func validateInput(s string, n int) (string, error) {
	var err1, err2 error
	if s == "" {
		err1 = errors.New("пустая строка")
	}
	if n < 0 {
		err2 = errors.New("отрицательное число")
	}
	err := errors.Join(err1, err2)
	if err != nil {
		return "", err
	}
	return "Ввод валиден", nil
}

func main() {
	var s string
	var n int
	s = ""
    n = -5
	message, err := validateInput(s, n)
	if err != nil {
		fmt.Printf("Ошибка: %v\n", err)
		return
	}
	fmt.Printf("Результат: %s\n", message)
}
```

**Вывод:**
```1c
Ошибка: пустая строка
отрицательное число
```

**Пояснение:** Функция `validateInput` проверяет строку и число, собирая ошибки в `err1` и `err2`. `errors.Join` объединяет их в одну ошибку, которая выводится в `main`.

#### **fmt.Errorf с %w**

Функция `fmt.Errorf` с использованием спецификатора `%w` позволяет оборачивать существующую ошибку, добавляя к ней контекст. Спецификатор `%w` сохраняет исходную ошибку, что позволяет проверять её с помощью `errors.Is` или `errors.As`. Это полезно, когда нужно уточнить причину ошибки, сохраняя доступ к исходной ошибке.

**Пример:**

```css
err := fmt.Errorf("ошибка ввода: %w", errors.New("пустая строка"))
```

Здесь `err` содержит сообщение "ошибка ввода: пустая строка", но исходная ошибка "пустая строка" доступна для проверки.

#### **Проверка обёрнутых ошибок**

Обёрнутые ошибки, созданные с `fmt.Errorf("%w")`, можно проверять с помощью `errors.Is` для определения исходной ошибки. Это позволяет реагировать на конкретные ошибки, даже если они обёрнуты дополнительным контекстом.

#### **Пример 2: Оборачивание ошибки с fmt.Errorf("%w")**

```go
package main

import (
	"errors"
	"fmt"
)

var ErrEmpty = errors.New("пустая строка")

func checkString(s string) (string, error) {
	if s == "" {
		return "", fmt.Errorf("ошибка ввода: %w", ErrEmpty)
	}
	return "Строка валидна", nil
}

func main() {
	var s string
	s = ""
	message, err := checkString(s)
	if err != nil {
		if errors.Is(err, ErrEmpty) {
			fmt.Println("Ошибка: пустая строка обнаружена")
		} else {
			fmt.Printf("Ошибка: %v\n", err)
		}
		return
	}
	fmt.Printf("Результат: %s\n", message)
}
```

**Вывод:**

```makefile
Ошибка: пустая строка обнаружена
```

**Пояснение:** Функция `checkString` возвращает ошибку, обёрнутую с помощью `fmt.Errorf("%w")`. В `main` `errors.Is` проверяет, содержит ли ошибка `ErrEmpty`, и выводит специальное сообщение.

#### **Пример 3: Комбинирование errors.Join и fmt.Errorf("%w")**

```go
package main

import (
	"errors"
	"fmt"
	"strconv"
)

var ErrZero = errors.New("нулевое значение")

func validateNumber(s string) (int, error) {
	var err1, err2 error
	if s == "" {
		err1 = errors.New("пустая строка")
	}
	n, err := strconv.Atoi(s)
	if err != nil {
		err2 = fmt.Errorf("ошибка парсинга: %w", err)
	} else if n == 0 {
		err2 = fmt.Errorf("невалидное число: %w", ErrZero)
	}
	err = errors.Join(err1, err2)
	if err != nil {
		return 0, err
	}
	return n, nil
}

func main() {
	var s string
	s = "0"
	result, err := validateNumber(s)
	if err != nil {
		if errors.Is(err, ErrZero) {
			fmt.Println("Ошибка: нулевое значение обнаружено")
		} else {
			fmt.Printf("Ошибка: %v\n", err)
		}
		return
	}
	fmt.Printf("Результат: %d\n", result)
}
```

**Вывод:**

```makefile
Ошибка: нулевое значение обнаружено
```

**Пояснение:** Функция `validateNumber` проверяет строку на пустоту и парсинг числа, оборачивая ошибки с `fmt.Errorf("%w")`. `errors.Join` объединяет все ошибки. В `main` `errors.Is` проверяет наличие `ErrZero`.

Ссылка на песочницу: [https://go.dev/play/p/a6FitPl6bJT](https://go.dev/play/p/a6FitPl6bJT)

#### **Рекомендации по использованию**

- Используйте `errors.Join` для объединения ошибок из нескольких проверок в одной функции.
- Применяйте `fmt.Errorf("%w")` для добавления контекста к существующей ошибке, сохраняя возможность её проверки.
- Проверяйте обёрнутые ошибки с `errors.Is` для обработки конкретных случаев.
- Убедитесь, что версия Go ≥ 1.20 для использования `errors.Join`.

#### **Преимущества и недостатки**

**Преимущества:**

- `errors.Join` позволяет сообщить о всех ошибках сразу.
- `fmt.Errorf("%w")` сохраняет исходную ошибку для анализа.
- Упрощает диагностику проблем за счёт контекста.

**Недостатки:**

- `errors.Join` требует Go 1.20 или новее.
- Оборачивание ошибок увеличивает сложность кода.
- Многословность при обработке нескольких ошибок.
