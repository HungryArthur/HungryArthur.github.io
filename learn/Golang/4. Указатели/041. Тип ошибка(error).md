Мы уже встречались с ошибками в предыдущих уроках, настало время разобраться что это и как работать с ними!

В Go ошибки — это способ сообщить о проблемах, которые могут возникнуть при выполнении программы, таких как неверный ввод или невозможность выполнить операцию. Ошибки обрабатываются явно, что делает код предсказуемым и надёжным. Стандартные функции в Go часто возвращают ошибки как дополнительное значение, чтобы вызывающий код мог проверить и обработать их. В этом шаге мы разберём, что такое ошибки, как они используются и как стандартные функции возвращают их.

#### **Что такое ошибка?**
В Go ошибка — это значение, которое описывает проблему, возникшую во время выполнения функции. Ошибки создаются с помощью пакета `errors` или функции `fmt.Errorf` и возвращаются как последнее значение в функциях, которые могут завершиться неудачей. Если ошибка отсутствует, возвращается `nil`. Вызывающий код должен проверить, равно ли значение ошибки `nil`, чтобы определить, произошла ли проблема.

Например, некоторая функция возвращает результат и ошибку:
```go
result, err := someFunction()
if err != nil {
    fmt.Println("Ошибка:", err)
    return
}
```

Если `err` не равно `nil`, программа обрабатывает ошибку, например, выводит её или прекращает выполнение.

#### **Как стандартные функции возвращают ошибки**
Многие функции в стандартной библиотеке Go возвращают ошибки в случае неудачи. Например:

- Пакет `strconv`: Функция `strconv.Atoi` преобразует строку в число и возвращает ошибку, если строка не является числом.
- Пакет `time`: Функция `time.Parse` возвращает ошибку, если строка не соответствует формату даты.

Ошибки, возвращаемые стандартными функциями, содержат текстовое описание проблемы, которое можно вывести или проанализировать. Вызывающий код обязан проверить возвращённую ошибку, чтобы избежать некорректной работы программы.

#### **Пример 1: Преобразование строки в число**
```go
package main

import (
	"fmt"
	"strconv"
)

func main() {
	number, err := strconv.Atoi("123")
	if err != nil {
		fmt.Println("Ошибка:", err)
		return
	}
	fmt.Println("Число:", number)
}
```
**Вывод:**
```makefile
Число: 123
```

**Пояснение:** Функция `strconv.Atoi` возвращает число и ошибку. Если строка корректна, ошибка равна `nil`, и программа выводит результат. Если строка не является числом, возвращается ошибка.

#### **Пример 2: Ошибка при неверной строке**
```go
package main

import (
	"fmt"
	"strconv"
)

func main() {
	number, err := strconv.Atoi("abc")
	if err != nil {
		fmt.Println("Ошибка:", err)
		return
	}
	fmt.Println("Число:", number)
}
```
**Вывод:**
```yaml
Ошибка: strconv.Atoi: parsing "abc": invalid syntax
```

**Пояснение:** Функция `strconv.Atoi` возвращает ошибку, так как строка "abc" не является числом. Программа проверяет `err` и выводит сообщение об ошибке.

#### **Пример 3: Проверка времени**
```go
package main

import (
	"fmt"
	"time"
)

func main() {
	t, err := time.Parse("2006-01-02", "2023-13-01")
	if err != nil {
		fmt.Println("Ошибка:", err)
		return
	}
	fmt.Println("Дата:", t)
}
```
**Вывод:**
```applescript
Ошибка: parsing time "2023-13-01" as "2006-01-02": cannot parse "01" as "02"
```

**Пояснение:** Функция `time.Parse` возвращает ошибку, если формат даты неверный. Программа проверяет `err` и выводит сообщение.

#### **Когда использовать проверку ошибок**
Ошибки следует проверять всегда, когда функция возвращает значение типа `error`. Игнорирование ошибок может привести к некорректной работе программы. Например, если не проверить ошибку от `strconv.Atoi`, программа может использовать неверное значение.

**Рекомендации:**
- Всегда проверяйте `err != nil` после вызова функции, возвращающей ошибку.
- Обрабатывайте ошибки сразу, чтобы избежать накопления проблем.
- Используйте описательные сообщения об ошибках для упрощения отладки.

**Преимущества ошибок:**
- Явная обработка ошибок делает код прозрачным и предсказуемым.
- Ошибки из стандартных функций содержат понятные сообщения.
- Простота проверки ошибок через `if err != nil`.

**Недостатки ошибок:**
- Многословный код из-за постоянной проверки ошибок.
- Необходимость вручную обрабатывать каждую ошибку.
