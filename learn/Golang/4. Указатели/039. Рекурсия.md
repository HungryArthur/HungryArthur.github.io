Рекурсия — это такое поведение функции, при котором она вызывает сама себя, чтобы решить задачу. Представьте, что у вас есть большая задача, которую можно разделить на меньшие, похожие задачи. Вместо того чтобы решать всё сразу, функция решает маленький кусочек и вызывает себя для оставшейся части.

Чтобы рекурсия работала правильно, нужен **базовый случай** — это момент, когда функция перестаёт вызывать себя и возвращает результат. Без базового случая рекурсия будет работать бесконечно и вызовет ошибку (переполнение стека).

**Разберем простой пример:**
Пусть, у вас есть куча конфет, и вам требуется посчитать все конфеты в ней.

**Рекурсивный процесс:**
1. Вы смотрите на кучу и говорите: "Сейчас я посчитаю одну конфету (убираете её), а остальные доверю этому же методу"
2. Повторяете: "1 + количество конфет в оставшейся куче"
3. Когда приходите к последней конфете, говорите: "Эта одна конфета — и есть ответ"

**На примере 3 конфет:**

- "1 + (количество конфет в куче из 2)"
- "1 + (1 + (количество конфет в куче из 1))"
- "1 + (1 + (1))" ← базовая ситуация (1 конфета)
- Возвращаемся назад: 1 + (1 + 1) = 3

Каждый вызов "помнит" своё число и ждёт результата от следующего, складывая их в обратном порядке — это и есть стек вызовов в действии.

Разберем теперь как это реализуется в коде.

В Go рекурсия выглядит как обычный вызов функции, только функция зовёт себя. Она удобна для задач, где можно разбить проблему на похожие части, но может быть медленной для больших данных, так как каждый вызов занимает место в памяти.

#### **Как работает рекурсия**

Когда функция вызывает себя, она:
1. Сохраняет текущую задачу в памяти (в стеке вызовов).
2. Решает меньшую задачу, вызывая себя с новыми данными.
3. Достигает базового случая и возвращает результат.
4. Возвращается назад, собирая результаты всех вызовов.

Например, для вычисления суммы чисел от 1 до 5 рекурсия работает так: считает 5 + (сумма от 1 до 4), затем 4 + (сумма от 1 до 3), и так до 1. Базовый случай — когда доходим до 0 или 1.

**Пример 1: Сумма чисел от 1 до n**
Эта программа вычисляет сумму чисел от 1 до введённого числа n.

```go
package main

import "fmt"

func sumToN(n int) int {
	if n <= 0 { // Базовый случай
		return 0
	}
	return n + sumToN(n-1) // Рекурсивный вызов
}

func main() {
	var n int
	fmt.Scan(&n)
	result := sumToN(n)
	fmt.Printf("Сумма: %d\n", result)
}
```

**Ввод:**
```undefined
4
```
**Вывод:**
```makefile
Сумма: 10
```

Пояснение: Для n=4 функция работает так:

- `sumToN(4) = 4 + sumToN(3)`
- `sumToN(3) = 3 + sumToN(2)`
- `sumToN(2) = 2 + sumToN(1)`
- `sumToN(1) = 1 + sumToN(0)`
- `sumToN(0) = 0` (базовый случай)

Итог: 0 → 1 → 1+2=3 → 3+3=6 → 6+4=10. Сумма: 1+2+3+4=10.

**Пример 2: Количество цифр в числе**
Эта программа считает, сколько цифр в числе, деля его на 10, пока не останется 0.
```go
package main

import "fmt"

func countDigits(n int) int {
    if n == 0 { // Базовый случай
        return 0
    }
    return 1 + countDigits(n/10) // Рекурсивный вызов
}

func main() {
    var n int
    fmt.Scan(&n)
    result := countDigits(n)
    fmt.Printf("Цифр: %d\n", result)
}
```
**Ввод:**
```undefined
123
```
**Вывод:**
```makefile
Цифр: 3
```

**Пояснение:** Для n=123 функция работает так:
- `countDigits(123) = 1 + countDigits(12)`
- `countDigits(12) = 1 + countDigits(1)`
- `countDigits(1) = 1 + countDigits(0)`
- `countDigits(0) = 0` (базовый случай)

Итог: 0 → 1 → 1+1=2 → 2+1=3. Число 123 имеет 3 цифры.

**Пример 3: Ошибка в базовом случае. Бесконечная рекурсия.**
Эта программа пытается считать, сколько цифр в числе, деля его на 10, пока не останется 0, но в ней нет базового случая, поэтому программа выполняется бесконечно, пока не случится переполнение стека.

```go
package main

import "fmt"

func countDigits(n int) int {
	return 1 + countDigits(n/10) // Рекурсивный вызов
}

func main() {
	var n = 100
	result := countDigits(n)
	fmt.Printf("Цифр: %d\n", result)
}
```

**Вывод:**
```iecst

Program exited.
```

**Пояснение:** Программа выполняется, но спустя некоторое время выполняет свое завершение, ничего не выводя при этом.
