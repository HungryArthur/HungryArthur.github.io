На самом деле есть несколько различных видов рекурсии.

Разберём следующие виды рекурсии: **косвенная**, **древовидная**, **хвостовая**. 

#### Косвенная рекурсия
Косвенная рекурсия — это такая рекурсия, когда функция A вызывает функцию B, а функция B вызывает A, создавая цикл вызовов. Это отличается от прямой рекурсии, где функция вызывает себя напрямую. Косвенная рекурсия встречается реже и часто используется в задачах, где несколько функций работают вместе, например, при разборе выражений.

**Пример: Проверка чётности и нечётности числа**
```go
package main

import "fmt"

func isEven(n int) int {
    if n == 0 {
        return 1 // Базовый случай: 0 — чётное
    }
    return isOdd(n - 1)
}

func isOdd(n int) int {
    if n == 0 {
        return 0 // Базовый случай: 0 — не нечётное
    }
    return isEven(n - 1)
}

func main() {
    var n int
    fmt.Scan(&n)
    result := isEven(n)
    if result == 1 {
        fmt.Printf("%d — чётное\n", n)
    } else {
        fmt.Printf("%d — нечётное\n", n)
    }
}
```

**Ввод:**
```undefined
4
```
**Вывод:**
```undefined
4 — чётное
```

**Пояснение:** `isEven` вызывает `isOdd`, а `isOdd` вызывает `isEven`. Для n=4: `isEven(4) → isOdd(3) → isEven(2) → isOdd(1) → isEven(0) = 1`. Это косвенная рекурсия, так как функции чередуются.

#### Древовидная рекурсия
Древовидная рекурсия — это такая рекурсия, когда функция делает несколько рекурсивных вызовов, создавая структуру, похожую на дерево. Это подходит для задач, где подзадача делится на несколько меньших подзадач, например, числа Фибоначчи. Но она может быть медленной, так как вызывает много повторных вычислений.

#### Пример: Числа Фибоначчи
```go
package main

import "fmt"

func fib(n int) int {
    if n <= 1 {
        return n // Базовый случай
    }
    return fib(n-1) + fib(n-2)
}

func main() {
    var n int
    fmt.Scan(&n)
    result := fib(n)
    fmt.Printf("Фибоначчи %d: %d\n", n, result)
}
```
**Ввод:**
```undefined
5
```
**Вывод:**
```yaml
Фибоначчи 5: 5
```

**Пояснение:** Для n=5: `fib(5) = fib(4) + fib(3)`, `fib(4) = fib(3) + fib(2)`, и так далее. Это создаёт дерево вызовов: два рекурсивных вызова на каждом шаге.

#### Хвостовая рекурсия
Хвостовая рекурсия — это когда рекурсивный вызов — последнее действие в функции, и после него нет других операций. Это позволяет (в некоторых языках) оптимизировать рекурсию в цикл, но в Go такой оптимизации нет, и глубокая рекурсия может переполнить стек.

**Пример: Сумма от 1 до n**
```go
package main

import "fmt"

func tailSum(n, acc int) int {
    if n == 0 {
        return acc // Базовый случай
    }
    return tailSum(n-1, acc + n)
}

func main() {
    var n int
    fmt.Scan(&n)
    result := tailSum(n, 0)
    fmt.Printf("Сумма: %d\n", result)
}
```
**Ввод:**
```undefined
5
```
**Вывод:**
```makefile
Сумма: 15
```

**Пояснение:** Для n=5: `tailSum(5, 0) → tailSum(4, 5) → tailSum(3, 9) → tailSum(2, 12) → tailSum(1, 14) → tailSum(0, 15) = 15`. Рекурсивный вызов — последний, это хвостовая рекурсия.

#### Отладка с debug.PrintStack
Функция `debug.PrintStack` из пакета `runtime/debug` полезна для отладки рекурсии. Она показывает стек вызовов, помогая понять, как глубоко зашла рекурсия и где возникла ошибка.

**Пример с отладкой (хвостовая рекурсия):**
```go
package main
import (
    "fmt"
    "runtime/debug"
)

func tailSumDebug(n, acc int) int {
    if n == 2 {
        fmt.Println("Отладка стека на n=2")
        debug.PrintStack()
    }
    if n == 0 {
        return acc
    }
    return tailSumDebug(n-1, acc + n)
}

func main() {
    var n int
    fmt.Scan(&n)
    result := tailSumDebug(n, 0)
    fmt.Printf("Сумма: %d\n", result)
}
```
**Ввод:**
```undefined
5
```
**Вывод (примерный):**
```bash
Отладка стека на n=2
goroutine 1 [running]:
runtime/debug.Stack(0x0?, 0x0?, 0x0?)
    .../debug/stack.go:24 +0x65
main.tailSumDebug(0x2, 0xc)
    .../main.go:10 +0x85
main.tailSumDebug(0x3, 0x9)
    .../main.go:14 +0x65
main.tailSumDebug(0x4, 0x5)
    .../main.go:14 +0x65
main.tailSumDebug(0x5, 0x0)
    .../main.go:14 +0x65
main.main()
    .../main.go:19 +0x45
Сумма: 15
```

**Пояснение:** При n=2 вызывается `debug.PrintStack`, показывая стек: `main` → `tailSumDebug(5)` → `tailSumDebug(4)` → `tailSumDebug(3)` → `tailSumDebug(2)`.
