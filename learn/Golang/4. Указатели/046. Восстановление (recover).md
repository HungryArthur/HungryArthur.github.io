Восстановление (recover) — это механизм в Go, который позволяет **перехватить** панику, вызванную функцией `panic`, и предотвратить аварийное завершение программы. Функция `recover` возвращает значение, переданное в `panic` (или `nil`, если паники не было), и используется для обработки критических ошибок, чтобы программа могла продолжить работу.

Функция `recover` работает **только** внутри отложенных функций (`defer`), так как она должна вызываться после возникновения паники, но до завершения функции. Это делает `recover` аналогом блока catch в других языках, но в Go он используется редко и только для восстановления от критических ошибок, а не для обработки обычных ошибок, для которых предпочтительны возвращаемые значения `error`.

### **Пример 1: Перехват автоматической паники**
```go
package main

import (
	"fmt"
)

func safeGetRune(s string, idx int) rune {
	defer func() {
		if r := recover(); r != nil {
			fmt.Printf("Перехвачена паника: %v\n", r)
			return
		}
	}()
	return []rune(s)[idx]
}

func main() {
	result := safeGetRune("test", 10)
	fmt.Printf("Результат: %d\n", result)
}
```
**Вывод:**
```applescript
Перехвачена паника: runtime error: index out of range [10] with length 4
Результат: 0
```

**Пояснение:** Передача индекса, большего чем длина строки,  вызывает автоматическую панику. Функция `recover` в `defer` перехватывает её, выводит сообщение, и программа продолжает работу, возвращая 0.

#### **Пример 2: Перехват самостоятельно вызванной паники**
```go
package main

import "fmt"

func safeDivide(a, b int) (int, error) {
	defer func() {
		if r := recover(); r != nil {
			fmt.Printf("Перехвачена паника: %v\n", r)
		}
	}()
	if b == 0 {
		panic("Деление на ноль")
	}
	return a / b, nil
}

func main() {
	result, err := safeDivide(10, 0)
	if err != nil {
		fmt.Println(err)
	}
	fmt.Printf("Результат: %d\n", result)
}
```
**Вывод:**
```makefile
Перехвачена паника: Деление на ноль
Результат: 0
```

**Пояснение:** Функция `safeDivide` вызывает `panic` при b=0. В `defer` используется `recover` для перехвата паники, вывода сообщения и продолжения работы программы.

При возникновении паники Go начинает процесс "раскрутки" стека вызовов, последовательно выполняя все отложенные функции (`defer`), встреченные на пути. Каждый `defer` получает возможность перехватить текущую панику с помощью `recover()`.

Если паника не была перехвачена, она продолжает движение вверх по стеку до тех пор, пока не достигнет корня программы, что приводит к аварийному завершению. Если паника была перехвачена, дальше движение она не продолжает и необходимо явно передать панику снова, если необходимо обработать панику во всех `defer`.

#### **Пример 3: Множественные defer**, однократная обработка паники
```go
package main

import "fmt"

func testPanic() {
	defer func() {
		if r := recover(); r != nil {
			fmt.Printf("Восстановление 2: %v\n", r)
		}
	}()
	defer func() {
		if r := recover(); r != nil {
			fmt.Printf("Восстановление 1: %v\n", r)
		}
	}()
	panic("Первая паника")
}

func main() {
	testPanic()
}
```
**Вывод:**
```yaml
Восстановление 1: Первая паника
```

**Пояснение:** Функции `defer` выполняются в обратном порядке. Первая паника перехватывается первым `defer`, обрабатывается им, далее обработанная паника не передается по стеку вызовов дальше и не перехваченную вторым `defer`.

####  **Пример 4: Множественные defer**, многократная обработка паники
```go
package main

import "fmt"

func testPanic() {
	defer func() {
		if r := recover(); r != nil {
			fmt.Printf("Восстановление 2: %v\n", r)
		}
	}()

	defer func() {
		if r := recover(); r != nil {
			fmt.Printf("Восстановление 1: %v\n", r)
			panic("Вторая паника") // Эта паника перехватывается следующим defer
		}
	}()

	panic("Первая паника")
}

func main() {
	testPanic()
	fmt.Println("Выполнение программы после перехвата паники....")
}
```
**Вывод:**
```yaml
Восстановление 1: Первая паника
Восстановление 2: Вторая паника
Выполнение программы после перехвата паники....
```

**Пояснение:** Функции `defer` выполняются в обратном порядке. Первая паника перехватывается ближайшим `defer`, обрабатывается им, но затем создается вторая паника, которая перехватывается следующим `defer` в стеке. Каждая новая паника в `defer` обрабатывается следующим `defer` в цепочке, пока все они не будут перехвачены.
