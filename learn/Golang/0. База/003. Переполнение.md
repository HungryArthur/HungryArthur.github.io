При переводе из целочисленного в целочисленное возможно **переполнение**.

**Переполнение при целочисленных преобразованиях**

Переполнение возникает, когда значение исходной переменной превышает допустимый диапазон целевого типа. В отличие от некоторых других языков, Go **не генерирует ошибку** при переполнении, а просто обрезает значение по модулю.

**Переполнение** происходит из-за ограниченности представления чисел в памяти компьютера. Каждый тип данных (например, целое число) имеет фиксированный размер, который определяет, сколько битов выделено для хранения значения. В случае целых чисел со знаком, например, типа int в 32-битной системе, значение представляется в виде знакового двоичного числа. Это означает, что один бит используется для обозначения знака (положительное или отрицательное число), а остальные биты — для представления самого числа.

Когда мы достигаем максимального значения, которое может хранить данное целое число (например, 2,147,483,647 для 32-битного int) и прибавляем к нему 1, происходит переполнение. В двоичном представлении все биты числа максимальны (11111111 11111111 11111111 11111111). При прибавлении 1 последовательно происходят «переносы» единиц: последние нули становятся единицами, пока не встретится единица, которая превратится в ноль, и в итоге лишь первый бит станет нулем (при этом будет перезаписан знак). Таким образом, результат переполнения возвращается к минимальному значению для этого типа данных (например, -2,147,483,648), что и приводит к неожиданным результатам.

Поэтому важно учитывать возможное переполнение при выполнении арифметических операций и применять механизмы проверки или использовать типы данных с большими диапазонами, если это необходимо.

Примеры опасных ситуаций:

```go
package main

import "fmt"

func main() {
	// 1. Преобразование из большего типа в меньший
	var bigInt int32 = 32768 // Максимальное int16: 32767
	smallInt := int16(bigInt)
	fmt.Println(smallInt) // Выведет: -32768 (переполнение!)

	// 2. Преобразование беззнакового в знаковый
	var unsigned uint = 255
	signed := int8(unsigned)
	fmt.Println(signed) // Выведет: -1

	// 3. Потеря данных при uint → int
	var huge uint64 = 9223372036854775808 
	normal := int64(huge)// Максимальное int64: 9223372036854775807
	fmt.Println(normal) // Выведет: -9223372036854775808
}
```

Особое внимание уделяйте:

```go
// Опасные сценарии
negativeToUnsigned := uint(-1) // Получим 18446744073709551615 (для uint64)
```
