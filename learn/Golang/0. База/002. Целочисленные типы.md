**Целочисленные типы** в языке программирования Go представляют собой фундаментальную категорию данных для работы с целыми числами. В отличие от некоторых других языков, Go предлагает разнообразные целочисленные типы, позволяющие программисту точно контролировать размер числа и необходимость работы с знаковыми/беззнаковыми значениями.

Для эффективной работы с числовыми данными важно понимать различия между типами, так как неправильный выбор может привести к переполнению или неэффективному использованию памяти.

Рассмотрим основные целочисленные типы, доступные в языке Go:

| Тип          | Категория   | Описание                                                                          | Диапазон значений                                       |
| ------------ | ----------- | --------------------------------------------------------------------------------- | ------------------------------------------------------- |
| `uint8`      | Беззнаковые | 8-битные целые без знака                                                          | 0 до 255                                                |
| `uint16`     | Беззнаковые | 16-битные целые без знака                                                         | 0 до 65,535                                             |
| `uint32`     | Беззнаковые | 32-битные целые без знака                                                         | 0 до 4,294,967,295                                      |
| `uint64`     | Беззнаковые | 64-битные целые без знака                                                         | 0 до 18,446,744,073,709,551,615                         |
| `int8`       | Знаковые    | 8-битные целые со знаком                                                          | -128 до 127                                             |
| `int16`      | Знаковые    | 16-битные целые со знаком                                                         | -32,768 до 32,767                                       |
| `int32`      | Знаковые    | 32-битные целые со знаком                                                         | -2,147,483,648 до 2,147,483,647                         |
| `int64`      | Знаковые    | 64-битные целые со знаком                                                         | -9,223,372,036,854,775,808 до 9,223,372,036,854,775,807 |
| float32      | Знаковые    | 32-битные плавающий числа                                                         |                                                         |
| float64      | Знаковые    | 64-битные плавающий числа                                                         |                                                         |
| `byte`       | Псевдоним   | Синоним `uint8`                                                                   | 0 до 255                                                |
| `rune`       | Псевдоним   | Синоним `int32` (код Unicode)                                                     | -2,147,483,648 до 2,147,483,647                         |
| `complex64`  |             | множество всех комплексных чисел с действительной и мнимой частями типа `float32` |                                                         |
| `complex128` |             | множество всех комплексных чисел с действительной и мнимой частями типа `float64` |                                                         |


А теперь напишим программу, которая использует все эти целочисленные типы, доступные в языке Go:

```go
package main

import "fmt"

func main() {
    // Знаковые целочисленные типы
    var a int    // зависит от платформы (32 или 64 бита)
    var b int8   // от -128 до 127
    var c int16  // от -32768 до 32767
    var d int32  // от -2147483648 до 2147483647 (также известен как rune)
    var e int64  // от -9223372036854775808 до 9223372036854775807

    // Беззнаковые целочисленные типы
    var f uint     // зависит от платформы
    var g uint8    // от 0 до 255 (также известен как byte)
    var h uint16   // от 0 до 65535
    var i uint32   // от 0 до 4294967295
    var j uint64   // от 0 до 18446744073709551615

    fmt.Println(a, b, c, d, e, f, g, h, i, j)
}
```

Тип `int` и `uint` являются наиболее универсальными - их размер зависит от архитектуры процессора (32 или 64 бита, то есть либо синонимы int32 и uint32, либо синонимы int64 и uint64). Это сделано для обеспечения оптимальной производительности на разных платформах. Однако, когда требуется точный контроль над размером данных, следует использовать типы с явно указанным размером (int8, uint32 и др.).
