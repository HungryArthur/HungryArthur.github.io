Глобальные переменные в Go — это изменяемые значения, объявляемые вне функций в области видимости пакета. Они доступны во всех функциях пакета и могут использоваться для хранения состояния программы, например, счётчиков, настроек или данных, общих для нескольких функций. В отличие от констант, глобальные переменные можно изменять, но их использование требует осторожности, чтобы избежать непредсказуемого поведения. Подробнее о функциях и о пакетах будет рассказано далее в курсе.

#### Объявление и использование глобальных переменных

Глобальные переменные объявляются с ключевым словом `var` вне функций. Тип может быть указан явно или выведен автоматически. Глобальные переменные автоматически инициализируются нулевым значением их типа (`0` для `int`, `""` для `string`, `false` для `bool`).

**Пример 1: Простая глобальная переменная**

Объявляем глобальную переменную для отслеживания количества пользователей.

```go
package main

import "fmt"

var userCount int // Глобальная переменная без явной инициализации

func main() {
    message := fmt.Sprintf("Значение переменной до использования: %d", userCount)
    fmt.Println(message)
    userCount++
    userCount++
    message = fmt.Sprintf("Количество пользователей: %d", userCount)
    fmt.Println(message)
}
```

**Вывод:**

```undefined
Значение переменной до использования: 0
Количество пользователей: 2
```

**Комментарий:** Глобальная переменная `userCount` (тип `int`, начальное значение 0) изменяется в функции main и выводится после изменения.

Ссылка на песочницу: [https://go.dev/play/p/LHA_-nVzBzy](https://go.dev/play/p/LHA_-nVzBzy)

**Пример 2: Глобальная переменная с явным типом и инициализацией**

Объявляем глобальную переменную с явным типом для хранения названия приложения.

```go
package main

import "fmt"

var appName string = "МоёПриложение"

func main() {
    appName = "СуперПриложение"
    fmt.Println("Название приложения:", appName)
}
```

**Вывод:**

```undefined
Название приложения: СуперПриложение
```

**Комментарий:** Глобальная переменная `appName` (тип `string`) инициализируется значением и изменяется в функции `main`.

Ссылка на песочницу: [https://go.dev/play/p/xD2c_fLbuAh](https://go.dev/play/p/xD2c_fLbuAh)

**Групповое объявление глобальных переменных**

Несколько глобальных переменных, также как и константы, можно объявить в блоке `var`, что удобно для связанных значений, например, настроек программы.

**Пример 3: Групповое объявление**

```go
package main

import "fmt"

var (
    totalScore    int    = 0
    maxScore      int    = 100
    gameStatus    string = "Активна"
)

func main() {
    totalScore = 1
    maxScore = 1000
    gameStatus = "Завершена"
    message := fmt.Sprintf("Счёт: %d, Статус: %s", totalScore, gameStatus)
    fmt.Println(message)
}
```

**Вывод:**

```makefile
Счёт: 1, Статус: Завершена
```

**Комментарий:** Блок `var` группирует связанные глобальные переменные (`totalScore`, `maxScore`, `gameStatus`), которые используются и изменяются в функции `main`.

Ссылка на песочницу: [https://go.dev/play/p/_3MQ3AtOkq4](https://go.dev/play/p/_3MQ3AtOkq4)

**Интересный факт:** Глобальные переменные можно инициализировать с помощью блока кода(функции) `init()`, которая выполняется перед `main`. Например:

```go
package main

import "fmt"

var config string

func init() {
	config = "Инициализация завершена"
}

func main() {
	fmt.Println(config)
}
```

**Вывод:**

```undefined
Инициализация завершена
```

**Комментарий:** Блок кода(функция)`init` выполняется до функции `main` и инициализирует глобальную переменную, которая в последующем используется в `main`.

Это полезно для настройки глобальных переменных на основе внешних данных или логики.

Ссылка на песочницу: [https://go.dev/play/p/PzZ0jJ7Edpm](https://go.dev/play/p/PzZ0jJ7Edpm)

**Важно!**

- Глобальные переменные могут быть изменены из любой функции, что может привести к трудноотслеживаемым ошибкам, особенно в больших программах.
- Чрезмерное использование глобальных переменных ухудшает читаемость и тестируемость кода.
- Если глобальная переменная не инициализирована явно, она получает нулевое значение, что может привести к неожиданному поведению, если забыть это учесть.
- В отличие от локальных переменных, короткое объявление (`:=`) недоступно для глобальных переменных.

**Полезные советы:**

- Используйте глобальные переменные только для данных, которые действительно должны быть общими для всего пакета, например, счётчиков или конфигураций.
- Группируйте связанные глобальные переменные в блок `var` для улучшения читаемости.
- Избегайте частого изменения глобальных переменных в разных функциях, чтобы предотвратить побочные эффекты.
- Используйте `init()` для сложной инициализации глобальных переменных, если требуется логика.
