Как вы уже узнали, в Go для работы с числами используются два основных типа:

- `int` - целые числа, а также `uint` - целые положительные числа 
- `float64` - числа с плавающей запятой

Рассмотрим, основные стандартные математические действия, которые поддерживаются ими:

|Операция|Пример с int|Пример с float64|Результат для примера с int|Результат для примера с float64|
|---|---|---|---|---|
|+|5 + 3|2.5 + 1.3|8|3.8|
|-|10 - 4|5.7 - 2.1|6|3.6|
|*|6 * 7|1.5 * 2.0|42|3.0|
|/|8 / 2|9.0 / 2.0|4|4.5|
|%|10 % 3|-|1 (остаток от деления)|-|

**Важно:**

- Для `int` деление `/` возвращает только целую часть
- Операция `%` (остаток от деления) работает только с `int`

При выполнении математических операций важно помнить про **точность float64**

**Погрешность округления** в Go может возникать при работе с числами с плавающей запятой. Это связано с тем, что такие числа не всегда могут быть точно представлены в двоичной системе. Давайте рассмотрим пример:

Примеры использования:

```go
package main

import (
	"fmt"
)

func main() {
	a := 0.1
	b := 0.2
	sum := a + b

	fmt.Println(a)   // Ожидаем 0.10000000000000000000
	fmt.Println(b)   // Ожидаем 0.20000000000000000000
	fmt.Println(sum) // Ожидаем 0.30000000000000000000, 
		             // но будет выведено 0.30000000000000004
}
```

В этом примере, делая сложение 0.1 и 0.2, мы ожидаем, что результат будет равен 0.3. Однако по факту из-за особенностей представления чисел с плавающей запятой мы получаем немного другое значение (например, 0.30000000000000004). Таким образом, это иллюстрирует проблему с погрешностью округления.
