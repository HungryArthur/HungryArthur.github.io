В Go синонимы типов (type aliases) позволяют создавать альтернативное имя для существующего типа, делая код более читаемым и выразительным. Синонимы не создают новый тип, а лишь дают другое название уже существующему базовому типу, такому как `int`, `string` или `bool`.

Синонимы объявляются с помощью ключевого слова `type` и оператора `=`. Например, `type UserID = int` делает `UserID` синонимом для `int`. В отличие от создания нового типа (об этом вы узнаете далее в курсе), синоним полностью совместим с исходным типом, и значения можно использовать взаимозаменяемо без приведения.

Синонимы типов помогают сделать код понятнее, особенно в больших проектах, где важно подчеркнуть назначение переменных. Например, вместо `bool` можно использовать `IsActive` для обозначения статуса активности. Однако их следует использовать умеренно, чтобы не усложнять код без необходимости.

**Пример 1: Синоним для числового типа**
```go
package main

import "fmt"

type UserID = int

func main() {
	var id UserID = 123
	var anotherID int = id // Совместимы, так как UserID — это int
	fmt.Printf("ID пользователя: %d\n", anotherID)
}
```
**Вывод:**
```nginx
ID пользователя: 123
```

**Пояснение:** `UserID` — синоним для `int`. Переменная `id` типа `UserID` присваивается переменной типа `int` без приведения, так как это один и тот же тип. В `main` демонстрируется использование синонима для вывода идентификатора.

**Пример 2: Синоним для строкового типа**
```go
package main

import "fmt"

type Username = string

func main() {
	var user Username = "Alice"
	var regularString string = user // Совместимы
	fmt.Printf("Имя пользователя: %s\n", regularString)
}
```
**Вывод:**
```undefined
Имя пользователя: Alice
```

**Пояснение:** `Username` — синоним для `string`. Значение типа `Username` можно использовать как `string`, и наоборот. В `main` показано, как синоним используется для хранения и вывода имени пользователя.

**Пример 3: Синоним для булевого типа**

```go
package main

import "fmt"

type IsActive = bool

func main() {
	var status IsActive = true
	var regularBool bool = status // Совместимы
	fmt.Printf("Пользователь активен: %v", regularBool)
}
```
**Вывод:**
```bash
Пользователь активен: true
```

**Пояснение:** `IsActive` — синоним для `bool`. Переменная `status` типа `IsActive` совместима с `bool`, и в `main` используется для  вывода статуса пользователя.

**Пример 4: Использование синонимов для обработки ввода**
```go
package main

import "fmt"

type Score = int
type Passed = bool

func main() {
	var score Score
	var passed Passed
	fmt.Scan(&score)
	passed = score >= 60
	fmt.Printf("Оценка: %d, тест пройден: %v\n", score, passed)
}
```
**Вывод** (для ввода `75`):
```bash
Оценка: 75, тест пройден: true
```
**Вывод** (для ввода `50`):
```bash
Оценка: 50, тест пройден: false
```

**Пояснение:** `Score` — синоним для `int`, а `Passed` — для `bool`. В `main` считывается оценка, проверяется условие прохождения теста, и синонимы используются для улучшения читаемости кода.
