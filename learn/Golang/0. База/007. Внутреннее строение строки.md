Строка - последовательность чисел, где каждый байт или последовательность байтов соответствует определённому символу Unicode.

Когда мы работаем со строками, содержащими Unicode-символы (например, текст на русском или эмодзи), важно понимать, что по сути строка - просто последовательность чисел:

```go
package main

import "fmt"

func main() {
    s := "Привет 🚀"
    fmt.Println("Строка:", s)
    // Выводим байты строки
    fmt.Println("Байты:", []byte(s))
}
```

**Вывод:**
```makefile
Строка: Привет 🚀
Байты: [208 159 209 128 208 184 208 178 208 181 209 130 32 240 159 154 128]
```

При обращении к строке по индексу в Go мы получаем конкретный байт, а не целый символ, потому что строка в Go - это байтовая последовательность в кодировке UTF-8. В этой кодировке:

- Символы ASCII (английские буквы, цифры) занимают 1 байт
- Символы европейских языков (например, буквы с диакритикой) - обычно 2 байта
- Кириллические символы - 2 байта
- Иероглифы и сложные символы (эмодзи) - 3 или 4 байта

Когда мы запрашиваем `s[0]` для строки `"Привет"`, мы получаем первый байт (208), который является только частью символа 'П'. Полный символ 'П' в UTF-8 кодируется двумя байтами: [208, 159].

**Преобразование числовых кодов UTF-8 в байты:**

Каждый символ Unicode (кодовая точка, например, U+041F для 'П') кодируется в UTF-8 как последовательность от 1 до 4 байтов:

- 1 байт (0xxxxxxx): для U+0000–U+007F (ASCII).
- 2 байта (110xxxxx 10xxxxxx): для U+0080–U+07FF, включая кириллицу.
- 3 байта (1110xxxx 10xxxxxx 10xxxxxx): для U+0800–U+FFFF.
- 4 байта (11110xxx 10xxxxxx 10xxxxxx 10xxxxxx): для U+10000–U+10FFFF, включая эмодзи.

Кодовая точка преобразуется в биты, которые распределяются по этим шаблонам. Например, для 'П' (U+041F): код 0x041F (00000100 00011111) кодируется как 11010000 10111111 ([208, 159]).

**Как программа понимает количество байтов для символа:**

Первый байт указывает длину последовательности:

- Начинается с `0` — 1 байт.
- Начинается с `110` — 2 байта.
- Начинается с `1110` — 3 байта.
- Начинается с `11110` — 4 байта.

Последующие байты начинаются с `10`, что отличает их от начала нового символа. Программа читает первый байт, определяет длину и собирает нужное количество байтов.

Таким образом, прямое обращение по индексу к строке:

1. Не учитывает Unicode-природу строки
2. Может возвращать неполные или некорректные данные для много байтовых символов
3. Подходит только для работы с ASCII или когда нам действительно нужны отдельные байты

Для корректной работы с символами необходимо использовать либо преобразование в `[]rune`, либо итерацию через `range`, которые специально разработаны для корректной обработки UTF-8.
