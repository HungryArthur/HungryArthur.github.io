Лямбда-функции — это функции без имени, которые вы создаёте прямо там, где они нужны. В Go их называют **анонимными функциями**, и они начинаются с `func`, но без имени. Их можно сохранить в переменную, передать в другую функцию или сразу вызвать. Вы уже встречались с синтаксисом лямбда функции в [2 шаге](https://stepik.org/lesson/1813520/step/2?unit=1839296) этой главы, сейчас же мы разберем их более внимательно.

Лямбда-функции удобны, когда вам нужно сделать что-то простое один раз, и не хочется писать отдельную функцию. Они могут использовать переменные из окружающей программы (замыкание), что делает их мощными для динамических задач.

Но с лямбда-функциями нужно быть осторожным: если они слишком сложные, код может стать трудно читаемым. Лучше использовать их для коротких и понятных действий.

**Пример 1: Лямбда-функция в переменной**

```go
package main

import "fmt"

func main() {
    var multiply func(int, int) int = func(a, b int) int {
        return a * b
    }
    result := multiply(6, 2)
    fmt.Printf("Результат: %d\n", result)
}
```

Вывод:

```go
Результат: 12
```

Пояснение: Мы создаём лямбда-функцию, которая перемножает два числа, и сохраняем её в переменную `multiply`. Потом вызываем её как обычную функцию. Это показывает, как лямбда-функции можно использовать сразу на месте.

**Пример 2: Лямбда-функция как аргумент**

```go
package main

import "fmt"

func apply(op func(int, int) int, a, b int) int {
	return op(a, b)
}

func main() {
	result := apply(func(x, y int) int { return x - y }, 7, 3)
	fmt.Printf("Результат: %d\n", result)
}
```

**Вывод:**

```go
Результат: 4
```

**Пояснение**: Лямбда-функция, вычитающая числа, передаётся прямо в `apply`. Это удобно, потому что не нужно создавать отдельную функцию для разового действия.

**Пример 3: Захват переменных через замыкание**

```go
package main

import "fmt"

func main() {
	// Внешняя переменная, которая будет захвачена
	threshold := 10

	// Лямбда-функция, использующая threshold через замыкание
	var limitValue func(int) int = func(x int) int {
		if x > threshold {
			return threshold
		}
		return x
	}

	// Применяем функцию к разным значениям
	fmt.Printf("Результат для 5: %d\n", limitValue(5))
	fmt.Printf("Результат для 15: %d\n", limitValue(15))
}
```

**Вывод:**

```yaml
Результат для 5: 5
Результат для 15: 10
```

**Пояснение:** Лямбда-функция `limitValue` захватывает переменную `threshold` из внешнего контекста (замыкание). Она ограничивает входное значение `x`, возвращая либо `x`, если оно меньше или равно `threshold`, либо `threshold`, если больше. Это демонстрирует, как лямбда-функции могут использовать внешние переменные для создания гибкого поведения.

**Пример 4: Сложный пример с захватом переменных**

```go
package main

import "fmt"

func makeCounter(start int, step string) func() int {
	count := start
	if step == "double" {
		return func() int {
			count *= 2
			return count
		}
	} else if step == "increment" {
		return func() int {
			count++
			return count
		}
	} else {
		return func() int {
			count += 3
			return count
		}
	}
}

func main() {
	var command string
	var start, calls int
	fmt.Scan(&command, &start, &calls)

	counter := makeCounter(start, command)
	for i := 0; i < calls; i++ {
		result := counter()
		fmt.Printf("%d ", result)
	}
	fmt.Println()
}
```

**Вывод** (для ввода `double 2 3`):

```undefined
4 8 16
```

**Вывод** (для ввода `increment 5 3`):

```undefined
6 7 8
```

**Вывод** (для ввода `other 1 3`):

```undefined
4 7 10
```

**Пояснение:** Функция `makeCounter` возвращает **лямбда-функцию**, которая захватывает переменную `count` и изменяет её в зависимости от команды (`double` — удваивает, `increment` — увеличивает на 1, другое — увеличивает на 3). Лямбда-функция использует замыкание, чтобы "запомнить" текущее значение `count` между вызовами. Программа считывает команду, начальное значение и количество вызовов, затем применяет возвращённую функцию указанное количество раз.
