Шаблон **Producer-Consumer** (Производитель-Потребитель) разделяет задачи на производство данных (производители) и их обработку (потребитель). В Go каналы обеспечивают безопасную и удобную синхронизацию между горутинами. В предыдущих уроках мы возвращали данные через общую память (например, срез, защищённый мьютексом), но в реальном мире это менее удобно из-за сложностей синхронизации и риска ошибок. Каналы упрощают передачу данных и делают код чище. В этом шаге мы реализуем Producer-Consumer с небуферизированным каналом, предоставим минималистичную схему с горутинами и каналом, и объясним преимущества каналов.

#### Почему каналы лучше общей памяти?
В предыдущих уроках мы могли использовать общую память (например, срез с `sync.Mutex`) для передачи данных между горутинами. Однако:
- **Общая память** требует мьютексов для защиты от состояний гонки, что усложняет код. Забытый `Lock` или `Unlock` может привести к deadlock или панике.
- **Сложность отладки**: Синхронизация через мьютексы увеличивает риск ошибок, особенно в больших системах, где горутины активно пишут в общий ресурс.
- **Каналы** решают эти проблемы:
    - Встроенная синхронизация: небуферизированные каналы блокируют отправку и чтение, пока обе стороны не готовы.
    - Чистый код: каналы заменяют явные мьютексы, упрощая логику.
    - Безопасность: исключают состояния гонки без дополнительной синхронизации.
    - Лёгкость масштабирования: каналы легко интегрируются в сложные конвейеры обработки данных.

Каналы позволяют запускать параллельные действия (например, подсчёт статистики) и передавать результаты безопасно и понятно, что особенно важно в реальных проектах, где читаемость и надёжность кода критически важны.

Базовая схема взаимодействия следующая:

```rust
┌───────────────┐    ┌───────────────────┐    ┌───────────────┐
│ горутина 1    │ -> │ канал для общения │ -> │ горутина 2    │
└───────────────┘    └───────────────────┘    └───────────────┘
```

Одна горутина (например, Горутина 1) может записывать данные в канал, а другая (Горутина 2) — считывать их оттуда. Фактически, мы уже используем этот принцип, так как функция `main` выполняется как отдельная горутина. Любая функция, запущенная через инструкцию `go`, становится горутиной. Это открывает возможность комбинировать их, создавая цепочки или конвейеры из горутин-обработчиков, где данные последовательно передаются от одной к другой.

#### Пример: Подсчёт чётных чисел
Функция `countAllSum` принимает список чисел и количество производителей `k`. Каждый производитель считает чётные числа в своей части списка и отправляет результат в небуферизированный канал. Функция суммирует результаты и возвращает итог. Главная горутина выводит результат.
```go
package main

import (
	"fmt"
	"sync"
)

func countAllSum(numbers []int, k int) int {
	ch := make(chan int)
	var wg sync.WaitGroup
	
	// Делим список на k частей
	chunkSize := (len(numbers) + k - 1) / k
	wg.Add(k)
	for i := 0; i < k; i++ {
		start := i * chunkSize
		end := start + chunkSize
		if end > len(numbers) {
			end = len(numbers)
		}
		if start < len(numbers) {
            // Для каждой части списка запускаем свою горутину с подсчетом
			go func(start, end int) {
				defer wg.Done()
				count := 0
				for j := start; j < end; j++ {
					if numbers[j]%2 == 0 {
						count++
					}
				}
				ch <- count // Производитель отправляет статистику
			}(start, end)
		} else {
			wg.Done()
		}
	}
	
	// Закрываем канал
	go func() {
		wg.Wait()
		close(ch)
	}()
	
	// Потребитель: суммируем статистику, обходя канал с результатами от всех горутин
	total := 0
	for count := range ch {
		total += count
	}
	
	return total
}

func main() {
	var n, k int
	fmt.Scan(&n)
	
	numbers := make([]int, n)
	for i := 0; i < n; i++ {
		fmt.Scan(&numbers[i])
	}
	fmt.Scan(&k)
	
	result := countAllSum(numbers, k)
	fmt.Printf("Even numbers: %d\n", result)
}
```

**Пояснение:**
- **Ввод**: `n = 6`, числа [1, 2, 3, 4, 5, 6], `k = 2`.
- **Работа функции**:
    - Список делится: [1, 2, 3] (1 чётное: 2) и [4, 5, 6] (2 чётных: 4, 6).
    - Производители отправляют статистику (1 и 2) в канал.
    - `countAllSum` суммирует (1 + 2 = 3) и возвращает.
- **Вывод**: `Even numbers: 3`.
- **Синхронизация**: Канал блокирует до чтения.
- **Избежание deadlock**: `for ... range` читает `k` значений, канал закрывается после производителей.
