**Что такое горутины?**

**Горутины** в Go — это лёгкие потоки выполнения, управляемые **рантаймом** Go(встроенная система времени выполнения, которая автоматически включается в каждую Go-программу и управляет ключевыми аспектами выполнения), а не операционной системой. Они позволяют запускать функции конкурентно, что делает их подходящими для параллельных задач, таких как, например, обработка независимых вычислений для ускорения выполнения программы. Горутины запускаются с помощью ключевого слова `go`, которое добавляет задачу в очередь планировщика Go.

**Пример 1: Простой запуск функций без горутин (последовательное выполнение)**
Когда функции вызываются без `go`, они выполняются последовательно в главной горутине (функция `main`также является горутиной). Это гарантирует, что каждая функция завершится перед вызовом следующей.
```go
package main

import (
	"fmt"
	"time"
)

func task(id int) {
	fmt.Printf("Task %d started\n", id)
	time.Sleep(time.Second) // Имитация работы
	fmt.Printf("Task %d finished\n", id)
}

func main() {
	for i := 1; i <= 3; i++ {
		task(i)
	}
	fmt.Println("Main completed")
}
```

**Пошаговое выполнение:**
- **Шаг 1:** Вызывается `task(1)`. Вывод: `Task 1 started`.
- **Шаг 2:** `task(1)` спит 1 секунду, затем выводит: `Task 1 finished`.
- **Шаг 3:** Вызывается `task(2)`. Вывод: `Task 2 started`.
- **Шаг 4:** `task(2)` спит 1 секунду, затем выводит: `Task 2 finished`.
- **Шаг 5:** Вызывается `task(3)`. Вывод: `Task 3 started`.
- **Шаг 6:** `task(3)` спит 1 секунду, затем выводит: `Task 3 finished`.
- **Шаг 7:** Вывод: `Main completed`.

**Вывод:**
```iecst
Task 1 started
Task 1 finished
Task 2 started
Task 2 finished
Task 3 started
Task 3 finished
Main completed
```

Функции выполняются в главной горутине последовательно, и каждая функция `task` полностью завершается (включая задержку в 1 секунду) перед вызовом следующей. Программа не завершается, пока все вызовы `task` не выполнятся, так как `main` ждёт их завершения. Общее время выполнения — около 3 секунд, так как задачи выполняются одна за другой.

**Пример 2: Простой запуск горутин в main без синхронизации**
Когда горутины запускаются в функции `main` с помощью `go`, они выполняются конкурентно, но главная горутина (функция `main`) не ждёт их завершения. Это может привести к тому, что программа завершится до выполнения горутин.
```go
package main

import (
	"fmt"
	"time"
)

func task(id int) {
	fmt.Printf("Task %d started\n", id)
	time.Sleep(time.Second) // Имитация работы
	fmt.Printf("Task %d finished\n", id)
}

func main() {
	for i := 1; i <= 3; i++ {
		go task(i)
	}
	fmt.Println("Main completed")
}
```

**Пошаговое выполнение:**
- **Шаг 1:** В цикле запускаются три горутины: `go task(1)`, `go task(2)`, `go task(3)`. Каждая добавляется в очередь планировщика Go.
- **Шаг 2:** Главная горутина выводит `Main completed` почти сразу, так как запуск горутин не блокирует выполнение.
- **Шаг 3:** Программа завершается, так как главная горутина закончила выполнение. Горутины `task` могут не успеть начать или завершить выполнение из-за быстрого завершения программы.

**Вывод (возможный):**
```css
Main completed
```

**Почему горутины могут не отработать?** Функция `main` является главной горутиной, и её завершение приводит к завершению всей программы, прерывая все остальные горутины. Планировщик Go не успевает выделить время для выполнения задач `task`, особенно если они включают задержки (например, `time.Sleep`). Без механизма синхронизации горутины могут не успеть вывести свои сообщения.

**Пример 3: Запуск горутин с time.Sleep**
Чтобы дать горутинам время на выполнение, можно использовать `time.Sleep` в `main`. Это не идеальный способ синхронизации, но он позволяет горутинам завершиться.
```go
package main

import (
	"fmt"
	"time"
)

func task(id int) {
	fmt.Printf("Task %d started\n", id)
	time.Sleep(time.Second) // Имитация работы
	fmt.Printf("Task %d finished\n", id)
}

func main() {
	for i := 1; i <= 3; i++ {
		go task(i)
	}
	time.Sleep(2 * time.Second) // Ждём 2 секунды
	fmt.Println("Main completed")
}
```

**Пошаговое выполнение:**
- **Шаг 1:** Запускаются три горутины: `go task(1)`, `go task(2)`, `go task(3)`. Каждая начинает выполнение и выводит `Task X started` (порядок может варьироваться).
- **Шаг 2:** Горутины спят 1 секунду одновременно (параллельно), затем выводят `Task X finished`.
- **Шаг 3:** Главная горутина ждёт 2 секунды с помощью `time.Sleep`, что достаточно для завершения всех горутин.
- **Шаг 4:** Главная горутина выводит `Main completed`.

**Вывод (порядок сообщений от горутин может варьироваться):**
```iecst
Task 1 started
Task 2 started
Task 3 started
Task 1 finished
Task 2 finished
Task 3 finished
Main completed
```

**Почему теперь всё выводится?** `time.Sleep(2 * time.Second)` задерживает главную горутину, давая планировщику Go достаточно времени для выполнения всех горутин. Каждая горутина занимает около 1 секунды, а 2 секунды ожидания гарантируют их завершение. Однако `time.Sleep` — ненадёжный метод, так как время ожидания может быть недостаточным или избыточным в зависимости от задач.

**Ключевые моменты:**
- Горутины запускаются с `go`, позволяя функциям выполняться конкурентно.
- Без синхронизации главная горутина может завершиться раньше, прерывая другие горутины.
- `time.Sleep` в `main` даёт горутинам время на выполнение, но это не точный метод синхронизации.
- Горутины управляются планировщиком Go, который распределяет их по потокам ОС для конкуренции или параллелизма.
