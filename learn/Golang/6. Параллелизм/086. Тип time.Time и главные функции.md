Теперь изучим этот пакет глубоко и системно.

#### Пакет time — основа работы с датой и временем
Пакет `time` предоставляет всё необходимое для работы с моментами времени, длительностями, таймерами, тикерами и часовыми поясами. Он используется практически в каждом серьёзном приложении на Go, потому что очень многое в нашем мире завязано на времени.

#### Главный тип — time.Time
В Go момент времени представлен значением типа `time.Time`. Это **структура** (value type), которая, если говорить упрощенно, содержит два основных поля:
```go
type Time struct {
    // wall — содержит время в наносекундах
    wall uint64
    // loc — указатель на часовой пояс
    loc *Location
}
```
Благодаря полю `loc` значение `time.Time` всегда «знает», в каком часовом поясе его нужно отображать.

#### Нулевое значение time.Time
Нулевое значение типа `time.Time` — это момент `0001-01-01 00:00:00 +0000 UTC`. +0000 UTC указывает, что данное время по Гринвичу. Проверить его можно методом `IsZero()`:
```swift
package main

import (
    "fmt"
    "time"
)

func main() {
    var t time.Time
    fmt.Printf("Тип: %T\n", t)
    fmt.Printf("Нулевое значение: %v\n", t)
    fmt.Printf("Это нулевое время? %v\n", t.IsZero())
}
```
**Вывод:**
```iecst
Тип: time.Time
Нулевое значение: 0001-01-01 00:00:00 +0000 UTC
Это нулевое время? true
```

Разберём эту строку по частям, чтобы всё стало понятно:
```yaml
0001-01-01 00:00:00 +0000 UTC
│         │         │       │
│         │         │       ├─ UTC — это всемирное координированное время (Coordinated Universal Time)
│         │         │       └─ +0000 — смещение от UTC: ноль часов, ноль минут
│         │         └─────────── 00:00:00 — полночь (начало суток)
│         └───────────────────── 01 января 1 года нашей эры
└─────────────────────────────── Полная дата и время
```

#### Почему именно +0000 UTC?
- **UTC** — это современный стандарт мирового времени. Это то же самое, что раньше называлось «время по Гринвичу» (GMT).
- **+0000** означает: «это время в часовом поясе, где нет никакого смещения от UTC».

То есть `+0000 UTC` = «время по Гринвичу, без коррекции на летнее/зимнее время». Если вы живёте в Москве, то ваше местное время сейчас, например, `+0300 MSK` — то есть на 3 часа впереди UTC.

#### Простая аналогия
Представьте, что UTC — это «нулевой меридиан» на глобусе, который проходит через Лондон (Гринвичскую обсерваторию). Все остальные часовые пояса отсчитываются от него:
- Москва — UTC+3 → `+0300 MSK`
- Нью-Йорк — UTC-5 → `-0500 EST`
- Токио — UTC+9 → `+0900 JST`
- А нулевое значение `time.Time` живёт прямо на этом «нулевом меридиане» — в `+0000 UTC`.

#### Как получить текущее время: time.Now()
Самая часто используемая функция в пакете — `time.Now()`. Она возвращает текущий момент времени в локальном часовом поясе системы.
```css
package main

import (
    "fmt"
    "time"
)

func main() {
    now := time.Now()
    fmt.Println("Текущее время:")
    fmt.Println(now)
}
```
**Пример вывода:**
```yaml
Текущее время:
2025-12-06 00:04:46.293 +0300 UTC+3 m=+0.000400129
```

#### Разбор вывода time.Time
- `2025-12-06 00:04:46.293` — дата и время с наносекундной точностью
- `+0300` — смещение от UTC (здесь +3 часа)
- `MSK` — аббревиатура часового пояса (Moscow Time)
- `m=+0.000400129` — монотонное время. Подробнее о том, что это, можно прочитать в [этой](https://stepik.org/lesson/2098911/step/%D0%9D%D0%B0%D0%BD%D0%B5%D1%81%D0%B5%D0%BA%D1%83%D0%BD%D0%B4%D1%8B%20%D1%81%20%D0%BD%D0%B0%D1%87%D0%B0%D0%BB%D0%B0%20%D1%8D%D0%BF%D0%BE%D1%85%D0%B8%20Unix) статье

В отличие от нулевого значения времени (`0001-01-01 00:00:00 +0000 UTC`), которое имеет точность только до секунд, реально измеренное время с помощью `time.Now()` включает наносекундную точность (отображаются миллисекунды: `00:04:46.293`) и содержит параметр монотонного времени `m=+0.000400129`, который указывает количество секунд, прошедшее с момента инициализации объекта времени (обычно с запуска программы или последнего вызова `time.Now()`).

#### Вывод отдельных компонентов времени
Структура `time.Time` предоставляет комплексные методы для извлечения всех компонентов временной метки: от макроэлементов (год, месяц, день) до микроэлементов (наносекунды) с учётом часового пояса (`Location`). 
```css
package main

import (
	"fmt"
	"time"
)

func main() {
	now := time.Now()

	fmt.Println("Текущее время:", now)
	fmt.Println("Год:", now.Year())
	fmt.Println("Месяц:", now.Month(), "(номер:", int(now.Month()), ")")
	fmt.Println("День месяца:", now.Day())
	fmt.Println("День недели:", now.Weekday())
	fmt.Println("Часы (24ч):", now.Hour())
	fmt.Println("Минуты:", now.Minute())
	fmt.Println("Секунды:", now.Second())
	fmt.Println("Наносекунды:", now.Nanosecond())
	fmt.Println("Часовой пояс:", now.Location())
}
```
**Пример вывода:**
```makefile
Текущее время: 2025-12-06 00:12:26.066874 +0300 MSK m=+0.000041126
Год: 2025
Месяц: December (номер: 12 )
День месяца: 6
День недели: Saturday
Часы (24ч): 0
Минуты: 12
Секунды: 26
Наносекунды: 66874000
Часовой пояс: Local
```


**Сравнение моментов времени**
Также структура `time.Time` реализует полный набор операций сравнения через встроенные методы: `Equal()`, `Before()`, `After()`
```css
package main

import (
    "fmt"
    "time"
)

func main() {
    t1 := time.Now()
    time.Sleep(50 * time.Millisecond)
    t2 := time.Now()

    fmt.Printf("t1: %v\n", t1)
    fmt.Printf("t2: %v\n", t2)
    fmt.Println("t1 раньше t2?", t1.Before(t2))
    fmt.Println("t1 позже t2?", t1.After(t2))
    fmt.Println("t1 равен t2?", t1.Equal(t2))
}
```
**Пример вывода:**
```yaml
t1: 2025-12-06 00:16:46.344566 +0300 MSK m=+0.000062542
t2: 2025-12-06 00:16:46.395587 +0300 MSK m=+0.051082959
t1 раньше t2? true
t1 позже t2? false
t1 равен t2? false
```

#### Создание времени вручную: time.Date
Функция `time.Date` позволяет создать произвольный момент времени:
```go
package main

import (
    "fmt"
    "time"
)

func main() {
    t := time.Date(
        2023,           // год
        time.October,   // месяц — можно как int, так и time.Month!
        10,             // день месяца
        23, 0, 0, 0,    // час, минута, секунда, наносекунда
        time.UTC,       // часовой пояс
    )
    fmt.Println("Созданное время:", t)
}
```
**Вывод:** `2023-10-10 23:00:00 +0000 UTC`

Также бывает полезным, что time.Date автоматически нормализует некорректные значения.  
Например, 32 сентября станет 2 октября, 25 часов станет 1 час следующего дня:
```iecst
// Примеры нормализации:
t1 := time.Date(2023, time.September, 32, 0, 0, 0, 0, time.UTC)
// Результат: 2023-10-02 00:00:00 +0000 UTC (32 сентября → 2 октября)

t2 := time.Date(2023, time.December, 31, 25, 70, 90, 0, time.UTC)
// Результат: 2024-01-01 02:11:30 +0000 UTC
// 25 часов → +1 день, 1 час
// 70 минут → +1 час, 10 минут  
// 90 секунд → +1 минута, 30 секунд

// Это работает и в обратную сторону (отрицательные значения):
t3 := time.Date(2023, time.January, -1, 0, 0, 0, 0, time.UTC)
// Результат: 2022-12-31 00:00:00 +0000 UTC (0 января → 31 декабря)
```

#### Важно: месяц — лучше не int!
В Go месяц имеет собственный тип `time.Month` с константами от `time.January` (1) до `time.December` (12). Это защищает от ошибок подобного рода:
```iecst
// Правильно
time.Date(2025, time.December, 31, 23, 59, 59, 0, time.UTC)

// Не ошибка, но будет переполнение и переход к следующему году, может быть неожиданным поведением
time.Date(2025, 13, 1, 0, 0, 0, 0, time.UTC)

// Не ошибка, но есть шанс ошибиться, думая, что 0 месяц соответствует Январю
time.Date(2025, 0, 1, 0, 0, 0, 0, time.UTC)
```
