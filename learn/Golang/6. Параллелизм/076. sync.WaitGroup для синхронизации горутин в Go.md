В конкурентном программировании на Go горутины позволяют выполнять задачи параллельно, но часто требуется дождаться их завершения перед продолжением работы программы. Использование `time.Sleep` , как в примерах рнее, для этой цели ненадёжно, так как время выполнения горутин зависит от системы, нагрузки и количества задач. Если задать слишком короткое время, некоторые горутины могут не успеть завершиться, а слишком длинное время замедляет программу. В этом шаге мы рассмотрим `sync.WaitGroup`, примитив синхронизации в Go, который позволяет точно и эффективно дожидаться завершения всех горутин без использования `time.Sleep`.

**Что такое sync.WaitGroup?**

`sync.WaitGroup` — это механизм синхронизации, который используется для ожидания завершения набора горутин. Он работает как счётчик задач: вы указываете, сколько горутин нужно дождаться, и уменьшаете счётчик, когда каждая горутина завершает работу. Программа продолжает выполнение, только когда счётчик становится равным нулю.

**Основные методы:**
- `Add(delta int)` — увеличивает счётчик на указанное количество (обычно число горутин). Вызывается до запуска горутин.
- `Done()` — уменьшает счётчик на 1. Вызывается, когда горутина завершает работу.
- `Wait()` — блокирует выполнение, пока счётчик не станет равным 0.

**Принцип работы:**
- Перед запуском горутин вызывается `Add(n)`, где `n` — число горутин.
- Каждая горутина вызывает `Done()` по завершении, обычно через `defer` для гарантированного вызова.
- `Wait()` блокирует вызывающую горутину (обычно основную), пока все горутины не вызовут `Done()`. Если счетчик `WaitGroup` > 0 и не будет вызван в последствие `Done()`, то программа заблокируется в дедлоке.

**Преимущества над time.Sleep:**
- `time.Sleep` требует угадывания времени ожидания, что может привести к преждевременному завершению программы (потеря результатов) или ненужным задержкам.
- `sync.WaitGroup` точно отслеживает завершение горутин, продолжая выполнение, как только все задачи завершены, независимо от их времени выполнения.
- Это делает код надёжным и эффективным, особенно при большом числе горутин или непредсказуемой нагрузке.

**Особенности:**
- `WaitGroup` не защищает от гонок данных; для этого используйте `sync.Mutex` или каналы.
- Вызов `Done()` без соответствующего `Add()` или избыточные вызовы `Done()` вызывают панику.
- `Add()` можно вызывать динамически, но нужно избегать отрицательного счётчика.

`sync.WaitGroup` необходим в сценариях, где программа должна дождаться завершения всех параллельных задач, таких как обработка данных, выполнение вычислений или операции ввода-вывода. Без `WaitGroup` программа может завершиться до того, как горутины закончат работу, особенно при использовании `time.Sleep`, которое не гарантирует ожидания всех задач. `WaitGroup` обеспечивает точную синхронизацию, делая код устойчивым к изменениям в производительности системы.

Рассмотрим четыре примера, демонстрирующие, как работает `sync.WaitGroup`. 

**Пример 1: Простое ожидание горутин**
Запускаем 5 горутин, каждая выводит сообщение, и ждём их завершения.
```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var wg sync.WaitGroup
	wg.Add(5)
	for i := 1; i <= 5; i++ {
		go func(id int) {
			defer wg.Done()
			fmt.Printf("Goroutine %d finished\n", id)
		}(i)
	}
	wg.Wait()
	fmt.Println("All goroutines completed")
}
```
**Вывод (порядок строк случайный):**
```css
Goroutine 1 finished
Goroutine 3 finished
Goroutine 2 finished
Goroutine 5 finished
Goroutine 4 finished
All goroutines completed
```

**Пояснение:** `wg.Add(5)` устанавливает счётчик на 5. Каждая горутина вызывает `wg.Done()` через `defer`, сигнализируя о завершении. `wg.Wait()` блокирует основную горутину, пока все 5 горутин не завершатся, после чего выводится финальное сообщение. Без `WaitGroup`, программа могла бы завершиться до вывода сообщений, а `time.Sleep` потребовала бы угадывания времени ожидания.

**Пример 2: Параллельная обработка чисел**
Запускаем 1000 горутин для вычисления квадратов чисел и суммируем результаты с использованием мьютекса.
```go
package main

import (
	"fmt"
	"sync"
)

func main() {
	var wg sync.WaitGroup
	var mu sync.Mutex
	sum := 0
	wg.Add(1000)
	for i := 1; i <= 1000; i++ {
		go func(id int) {
			defer wg.Done()
			result := id * id
			mu.Lock()
			sum += result
			mu.Unlock()
		}(i)
	}
	wg.Wait()
	fmt.Printf("Total sum of squares: %d\n", sum)
}
```
**Вывод:**
```bash
Total sum of squares: 333833500
```

**Пояснение:** 1000 горутин вычисляют квадраты чисел и добавляют их к общей сумме, защищённой мьютексом для предотвращения гонок данных. `wg.Wait()` гарантирует, что сумма выводится только после завершения всех вычислений. Использование `time.Sleep` вместо `wg.Wait()` могло бы привести к выводу неверной суммы, если не все горутины успели завершиться.

**Важно!**
- `sync.WaitGroup` позволяет дожидаться завершения всех горутин без использования `time.Sleep`, обеспечивая точную синхронизацию.
- Вызовите `Add()` перед запуском горутин, `Done()` в каждой горутине (обычно через `defer`), и `Wait()` для блокировки.
- `WaitGroup` не защищает от гонок данных; используйте `sync.Mutex` или каналы для управления доступом к общим данным.
- Подходит для сценариев с параллельной обработкой данных, вычислениями или операциями ввода-вывода.
