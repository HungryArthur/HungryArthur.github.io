#### Простой if
Оператор `if` проверяет условие и выполняет блок кода, если оно истинно. Условие должно быть выражением типа `bool`, например, сравнение с помощью операторов `>`, `==` или `&&`.

**Пример 1: Проверка положительного числа**

Программа считывает число и проверяет, положительное ли оно. Если считывание вызывает ошибку, программа завершается.  Подробнее про ошибки будут рассказано в следующих уроках.
```go
package main

import "fmt"

func main() {
    // Считываем число
    var inputNumber int
    _, err := fmt.Scan(&inputNumber)
    if err != nil {
        fmt.Println("Error")
        return
    }
    
    // Проверяем, положительное ли число
    if inputNumber > 0 {
        fmt.Println("Positive")
    }
}
```
**Ввод:**
```undefined
42
```
**Вывод:**
```nginx
Positive
```
**Ввод:**
```no-highlight
-5
```
**Вывод:**
**Комментарий:** Если `inputNumber > 0`, выводится `"Positive"`. Если условие ложно, ничего не выводится.

#### else и else if

Для обработки альтернативных случаев используется `else` (выполняется, если условие `if` ложно) и `else if` (дополнительные условия). `else if` позволяет проверять несколько условий последовательно.

**Пример 2: Классификация числа**

Программа определяет, является ли число положительным, отрицательным или нулевым.
```go
package main

import "fmt"

func main() {
    // Считываем число
    var inputNumber int
    _, err := fmt.Scan(&inputNumber)
    if err != nil {
        fmt.Println("Error")
        return
    }
    
    // Классифицируем число
    message := "Zero"
    if inputNumber > 0 {
        message = "Positive"
    } else if inputNumber < 0 {
        message = "Negative"
    }
    
    fmt.Println(message)
}
```
**Ввод:**
```no-highlight
-10
```
**Вывод:**
```nginx
Negative
```

**Комментарий:** `else if` проверяет `inputNumber < 0` только если `inputNumber > 0` ложно. Если оба условия ложны, выводится `"Zero"`.

#### Инициализация в if
В Go можно объявить переменную внутри `if` перед условием, разделив её точкой с запятой (`;`). Это ограничивает область видимости переменной блоком `if`, что удобно для обработки ошибок или временных значений.

**Пример 3: Проверка ввода с инициализацией**

Программа считывает число и строку, проверяет, корректен ли ввод.
```go
package main

import "fmt"

func main() {
    var inputNumber int
    var inputString string
    if count, err := fmt.Scanln(&inputNumber, &inputString); err != nil || count != 2 {
        fmt.Println("Error")
        return
    }
    
    if inputNumber > 0 {
        message := fmt.Sprintf("Number: %d, String: %s", inputNumber, inputString)
        fmt.Println(message)
    } else {
        fmt.Println("Invalid number")
    }
}
```
**Ввод:**
```undefined
25 Hello
```
**Вывод:**
```javascript
Number: 25, String: Hello
```
**Ввод:**
```no-highlight
-5 World
```
**Вывод:**
```typescript
Invalid number
```

**Комментарий:** `count, err := fmt.Scan(...)` инициализирует переменные в `if`. Они видны только в блоке `if`. Если `inputNumber > 0`, выводится форматированная строка, иначе — `"Invalid number"`.

Вспомним о проблеме, которая возникала у нас в шаге, когда мы хотели **безопасно** обрезать строки. При обрезке строк с помощью срезов (например, `input[:n]`) необходимо **проверять** длину строки функцией `len()`, чтобы избежать паники (ошибки времени выполнения) при обращении к индексам за пределами строки.

**Пример кода:**
```go
package main

import "fmt"

func main() {
	input := "Hiiiii"
	maxLen := 3
	var result string
	if len(input) >= maxLen {
		result = input[:maxLen]
	} else {
		result = input
	}
	message := fmt.Sprintf("Результат: %s", result)
	fmt.Println(message)
}
```
**Вывод:**
```makefile
Результат: Hii
```

**Комментарий:** Проверяем длину строки `input` с помощью `len()`. Если длина ≥ `maxLen`, обрезаем до первых трёх символов, иначе оставляем строку без изменений.

**Совет:** Всегда проверяйте `len(input)` перед обрезкой, либо любой другой работой со строкой, где необходимы индексы строки, чтобы избежать паники при обращении к тем индексам, которые больше длины строки.

**Интересный факт**:
Инициализация в `if` (например, `if x := 5 * a; x != 10 {...}`) ограничивает область видимости переменной `x`, что делает код безопаснее и предотвращает случайное использование переменной вне блока.

**Важно!**
- Условие в `if` должно возвращать `bool`; выражения вроде `if 1 {}` вызовут ошибку компиляции, в отличие от некоторых других языков программирования.
- Переменные, объявленные в `if`, недоступны вне блока `if` или `else`.

#### Полезные советы
- Проверяйте ошибки с `if err != nil` сразу после операций, которые могут возвращать ошибки. Подробнее про ошибки будут рассказано в следующих уроках.
- Используйте `else if` для последовательной проверки условий.
- Применяйте инициализацию в `if` для временных переменных, например, для обработки ошибок.
