Для управления полями, особенно приватными (неэкспортируемыми), используются конструкторы, геттеры, сеттеры и методы. Эти механизмы обеспечивают инкапсуляцию, контроль доступа и удобное взаимодействие с данными.

**Конструкторы**

В Go нет встроенных конструкторов, как в C++ или Java. Вместо этого используются функции, создающие экземпляр структуры с заданными значениями. Такие функции обычно называют `NewИмяСтруктуры` (например, `NewStudent`). Конструкторы применяются для:

- **Инициализации полей:** Установка начальных значений, включая приватные поля, недоступные вне пакета.
- **Инкапсуляции:** Скрытие деталей реализации, обеспечивая корректное начальное состояние.
- **Упрощения создания:** Удобное создание структуры без необходимости задавать все поля вручную.
- **Контроля доступа:** Инициализация приватных полей, которые нельзя изменить напрямую вне пакета.

Конструкторы особенно полезны для приватных полей (с маленькой буквы), так как они недоступны для прямой инициализации вне пакета(подробнее о пакетах будет рассказано далее).

**Пример конструктора:**
```go
package main

import "fmt"

type Student struct {
	Name   string
	points int
}

func NewStudent(name string, points int) Student {
	return Student{
		Name:   name,
		points: points,
	}
}

func main() {
	s := NewStudent("Алекс", 85)
	fmt.Printf("Студент: %s, Баллы: %d\n", s.Name, s.points)
}
```
**Вывод:**
```makefile
Студент: Алекс, Баллы: 85
```

**Пояснение:** Функция `NewStudent` создаёт структуру `Student`, инициализируя экспортируемое поле `Name` и приватное поле `points`. Это позволяет корректно создать объект.

**Методы структур и типы ресиверов**

Методы — это функции, связанные со структурой через приёмник. В Go существуют два типа ресиверов:
- **`func (r Record)` (значение):** Работает с копией структуры. Изменения в методе не затрагивают оригинальную структуру, и метод часто возвращает новую структуру. Используется, когда изменения не нужны или структура небольшая.
- **`func (r *Record)` (указатель):** Работает с оригинальной структурой через указатель. Изменения в методе затрагивают исходную структуру, и возврат не требуется. Используется для модификации структуры или при работе с большими данными для экономии памяти.

**Зачем нужны:** Методы добавляют функциональность, связанную с данными структуры, например, изменение координат или вычисление значений.

**Применение:** Реализуют операции, такие как перемещение объекта или обновление состояния.

**Пример метода с разными типами ресиверов:**
```go
package main
import "fmt"

type Robot struct {
    X      int
    Y      int
    energy int
}

func NewRobot(x, y, energy int) Robot {
    return Robot{
        X:      x,
        Y:      y,
        energy: energy,
    }
}

func (r Robot) MoveNorthValue(dist int) Robot {
    r.Y += dist
    r.energy -= dist
    return r
}

func (r *Robot) MoveNorthPointer(dist int) {
    r.Y += dist
    r.energy -= dist
}

func main() {
    r1 := NewRobot(0, 0, 100)
    r2 := r1.MoveNorthValue(10)
    fmt.Printf("r1: X=%d, Y=%d, Энергия: %d\n", r1.X, r1.Y, r1.energy)
    fmt.Printf("r2: X=%d, Y=%d, Энергия: %d\n", r2.X, r2.Y, r2.energy)
    r3 := NewRobot(0, 0, 100)
    r3.MoveNorthPointer(10)
    fmt.Printf("r3: X=%d, Y=%d, Энергия: %d\n", r3.X, r3.Y, r3.energy)
}
```
**Вывод:**
```makefile
r1: X=0, Y=0, Энергия: 100
r2: X=0, Y=10, Энергия: 90
r3: X=0, Y=10, Энергия: 90
```

**Пояснение:** Метод `MoveNorthValue` с ресивером по значению работает с копией, поэтому `r1` не изменяется, а изменения возвращаются в `r2`. Метод `MoveNorthPointer` с указательным ресивером изменяет оригинальную структуру `r3` напрямую.

**Геттеры**

Геттеры — пример методов структуры, возвращающие значения полей структуры. Они используются для доступа к приватным полям. Геттеры обычно именуются как `GetИмяПоля` или `ИмяПоля`.
- **Зачем нужны:** Обеспечивают безопасный доступ к данным, скрывая внутреннее представление.
- **Применение:** Позволяют другим пакетам получать значения приватных полей через экспортируемые методы.

**Пример геттера:**
```go
package main

import "fmt"

type Student struct {
	Name   string
	points int
}

func NewStudent(name string, points int) Student {
	return Student{
		Name:   name,
		points: points,
	}
}

func (s Student) GetPoints() int {
	return s.points
}

func main() {
	s := NewStudent("Мария", 70)
	fmt.Printf("Студент: %s, Баллы: %d\n", s.Name, s.GetPoints())
}
```
**Вывод:**
```makefile
Студент: Мария, Баллы: 70
```

**Пояснение:** Метод `GetPoints` возвращает значение приватного поля `points`, обеспечивая доступ к данным, даже если прямой доступ ограничен.

**Сеттеры**

Сеттеры — пример методов структуры, изменяющие значения полей структуры. Они используются для обновления приватных полей с контролем над изменениями. Сеттеры обычно именуются как `SetИмяПоля`.

- **Зачем нужны:** Позволяют безопасно изменять приватные поля, скрывая прямой доступ.
- **Применение:** Устанавливают новые значения, часто с проверками, и могут использовать указательный ресивер для изменения оригинальной структуры.

**Пример сеттера с указательным ресивером:**
```go
package main

import "fmt"

type Student struct {
	Name   string
	points int
}

func NewStudent(name string, points int) Student {
	return Student{
		Name:   name,
		points: points,
	}
}

func (s *Student) SetPoints(newPoints int) {
	s.points = newPoints
}

func main() {
	s := NewStudent("Иван", 50)
	fmt.Printf("Студент: %s, Баллы: %d\n", s.Name, s.points)
	s.SetPoints(80)
	fmt.Printf("После изменения: Студент: %s, Баллы: %d\n", s.Name, s.points)
}
```
**Вывод:**
```makefile
Студент: Иван, Баллы: 50
После изменения: Студент: Иван, Баллы: 80
```

**Пояснение:** Метод `SetPoints` с указательным ресивером `*Student` изменяет оригинальную структуру, а не её копию. Это позволяет обновить приватное поле `points` без возврата новой структуры.
