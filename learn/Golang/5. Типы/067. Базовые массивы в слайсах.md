#### Как работают слайсы?

Слайс — это дескриптор, который указывает на участок базового массива. Когда вы создаёте слайс из массива или другого слайса, он наследует указатель на тот же базовый массив (или его часть). Это делает слайсы эффективными, так как они не копируют данные, а лишь предоставляют доступ к ним. Однако это также означает, что модификация элементов слайса изменяет базовый массив, что влияет на все слайсы, ссылающиеся на этот массив.

**Пример с общим базовым массивом:**
```go
package main

import "fmt"

func main() {
	arr := [5]int{1, 2, 3, 4, 5}
	s1 := arr[1:4] // Слайс: [2, 3, 4]
	s2 := arr[2:5] // Слайс: [3, 4, 5]
	fmt.Printf("s1: %v, s2: %v\n", s1, s2)
	s1[1] = 100 // Изменяем элемент s1[1] (это arr[2])
	fmt.Printf("После изменения s1: s1: %v, s2: %v\n", s1, s2)
	fmt.Printf("Базовый массив: %v\n", arr)
}
```
**Вывод:**
```yaml
s1: [2 3 4], s2: [3 4 5]
После изменения s1: s1: [2 100 4], s2: [100 4 5]
Базовый массив: [1 2 100 4 5]
```

**Пояснение:** Слайсы `s1` и `s2` ссылаются на один базовый массив `arr`. Изменение `s1[1]` (элемент 3 в массиве) меняет базовый массив, что отражается в `s2`, так как `s2` включает этот же элемент.

#### Модификация слайсов
Слайсы поддерживают динамическое изменение размера с помощью встроенной функции `append`. При добавлении элементов, если ёмкость слайса исчерпана, Go автоматически создаёт новый базовый массив с большей ёмкостью (обычно удваивая её) и копирует данные.

**Пример добавления элементов:**
```go
package main

import "fmt"

func main() {
	s := make([]int, 2, 4)
	s[0], s[1] = 1, 2
	fmt.Printf("Исходный слайс: %v, длина: %d, ёмкость: %d\n", s, len(s), cap(s))
	s = append(s, 3, 4)
	fmt.Printf("После append: %v, длина: %d, ёмкость: %d\n", s, len(s), cap(s))
	s = append(s, 5)
	fmt.Printf("После append с превышением ёмкости: %v, длина: %d, ёмкость: %d\n", s, len(s), cap(s))
}
```
**Вывод:**
```less
Исходный слайс: [1 2], длина: 2, ёмкость: 4
После append: [1 2 3 4], длина: 4, ёмкость: 4
После append с превышением ёмкости: [1 2 3 4 5], длина: 5, ёмкость: 8
```

**Пояснение:** Слайс создаётся с длиной 2 и ёмкостью 4. Первые два `append` добавляют элементы в пределах ёмкости. При добавлении пятого элемента ёмкость удваивается до 8.

#### Изменение всех слайсов, ссылающихся на один базовый массив
Поскольку слайсы ссылаются на один и тот же базовый массив, изменение элементов через один слайс (например, через индексацию `s[i] = x`) изменяет базовый массив, что отражается во всех слайсах, которые включают эти элементы. Однако, если `append` создаёт новый базовый массив (при превышении ёмкости), слайс начинает ссылаться на новый массив, и изменения в нём не затрагивают другие слайсы.

**Пример изменения общего базового массива:**
```go
package main

import "fmt"

func main() {
	arr := [5]int{1, 2, 3, 4, 5}
	s1 := arr[0:3] // [1, 2, 3]
	s2 := arr[1:5] // [2, 3, 4, 5]
	fmt.Printf("До изменения: s1: %v, s2: %v\n", s1, s2)
	s1[1] = 100 // Изменяем элемент 2 в базовом массиве
	fmt.Printf("После изменения s1: s1: %v, s2: %v\n", s1, s2)
	s2 = append(s2, 6) // s2 теперь ссылается на новый базовый массив
	s2[1] = 200        // Изменяем элемент в новом массиве s2
	fmt.Printf("После append и изменения s2: s1: %v, s2: %v\n", s1, s2)
	fmt.Printf("Первоначальный массив: %v", arr[:])
}
```
**Вывод:**
```less
До изменения: s1: [1 2 3], s2: [2 3 4 5]
После изменения s1: s1: [1 100 3], s2: [100 3 4 5]
После append и изменения s2: s1: [1 100 3], s2: [100 200 4 5 6]
Первоначальный массив: [1 100 3 4 5]
```

**Пояснение:** Изменение `s1[1]` меняет базовый массив, что отражается в `s2`. После `append` на `s2` создаётся новый базовый массив, и последующее изменение `s2[1]` не влияет на `s1`.

#### Копирование слайсов
Функция `copy(dst, src []T)` копирует элементы из одного слайса в другой, но не меняет базовый массив. Это позволяет создать независимую копию данных слайса.

**Пример копирования:**
```go
package main

import "fmt"

func main() {
	s1 := []int{1, 2, 3}
	s2 := make([]int, len(s1))
	copy(s2, s1)
	s1[0] = 100
	fmt.Printf("s1: %v, s2: %v\n", s1, s2)
}
```
**Вывод:**
```yaml
s1: [100 2 3], s2: [1 2 3]
```

**Пояснение:** `copy` создаёт независимую копию элементов, поэтому изменение `s1` не влияет на `s2`.

#### Передача слайсов в функции
Слайсы в Go имеют особенность: они передаются в функцию по значению, но это значение содержит указатель на базовый массив. Это означает, что хотя сама структура слайса (указатель, длина и вместимость) копируется, обе копии — и исходная, и переданная — ссылаются на один и тот же массив данных.

**Как следствие:**
- **Изменение элементов:** Любые модификации существующих элементов слайса внутри функции будут видны снаружи, так как изменяются данные в общем массиве.
- **Добавление элементов:** Новые, добавленные элементы, также не будут видны вызывающей стороне, поскольку переменные, хранящие размер базового массива передаются по копии и не изменяются после добавления новых элементов в функции.
- **Операции изменения длины:** Если внутри функции происходит добавление элементов (`append`), которое приводит к перевыделению памяти (когда `cap` недостаточен), слайс начинает указывать на _новый_ базовый массив. В этом случае все последующие изменения элементов (в новом массиве) уже не будут отражаться на исходном слайсе.

Это поведение требует особой аккуратности, чтобы избежать трудноуловимых ошибок.

Слайсы передаются без указания размера: `func process(s []int)`.

**Передача переменного числа аргументов:** В Go поддерживаются функции с переменным числом аргументов (variadic functions) с использованием синтаксиса `...`, например, `func process(nums ...int)`. Такие функции принимают произвольное количество аргументов указанного типа, которые внутри функции обрабатываются как слайс. Это удобно для обработки наборов данных, как описано в уроке [Передача переменного числа аргументов в функции](https://stepik.org/lesson/1813519/step/12?unit=1839295). Например, можно передать отдельные элементы или целый слайс, используя оператор `...`.

**Пример передачи массива и слайса в функции:**
```go
package main

import "fmt"

func processSlice(s []int) {
	s[0] = 100 // Изменяет базовый массив
	fmt.Printf("Внутри функции (слайс): %v\n", s)
}

func sumVariadic(nums ...int) int {
	total := 0
	for _, num := range nums { // nums обрабатывается как слайс
		total += num
	}
	return total
}

func main() {
	s := []int{1, 2, 3}

	fmt.Printf("Исходный слайс: %v\n", s)
	processSlice(s)
	fmt.Printf("После функции (слайс): %v\n", s)

	// Использование variadic функции
	fmt.Printf("Сумма через variadic (отдельные элементы): %d\n", sumVariadic(1, 2, 3))
	fmt.Printf("Сумма через variadic (слайс): %d\n", sumVariadic(s...))
}
```
**Вывод:**
```less
Исходный слайс: [1 2 3]
Внутри функции (слайс): [100 2 3]
После функции (слайс): [100 2 3]
Сумма через variadic (отдельные элементы): 6
Сумма через variadic (слайс): 105
```

**Пояснение:**  Функция `processSlice` изменяет элементы базового массива через слайс. Функция `sumVariadic` принимает переменное число аргументов, которые обрабатываются как слайс. Оператор `...` позволяет передать слайс `s` как отдельные элементы.

#### Использование оператора `...` с `append`
Оператор `...` в Go также используется с функцией `append` для добавления всех элементов одного слайса к другому. Это позволяет "распаковывать" слайс в отдельные элементы, которые затем добавляются к целевому слайсу. Синтаксис: `slice = append(slice, anotherSlice...)`.

**Пример добавления слайса с помощью `...`:**
```go
package main

import "fmt"

func main() {
	s1 := []int{1, 2, 3}
	s2 := []int{4, 5, 6}
	s1 = append(s1, s2...) // Распаковываем s2
	fmt.Printf("После append: %v\n", s1)
}
```
**Вывод:**
```go
После append: [1 2 3 4 5 6]
```

**Пояснение:** Оператор `...` распаковывает слайс `s2` в отдельные элементы (4, 5, 6), которые добавляются к `s1` с помощью `append`. Это эквивалентно `append(s1, 4, 5, 6)`.

#### Особенности слайсов
- **Динамический размер:** Слайсы позволяют добавлять элементы через `append`, в отличие от массивов с фиксированным размером.
- **Общий базовый массив:** Несколько слайсов, созданных из одного массива, могут делить общие данные, что требует осторожности при модификации.
- **Нулевой слайс:** `var s []int` создаёт слайс с `len=0`, `cap=0` и значением `nil`, но он готов к использованию с `append`.
- **Копирование:** Функция `copy(dst, src []T)` копирует элементы из одного слайса в другой, но не меняет базовый массив.
