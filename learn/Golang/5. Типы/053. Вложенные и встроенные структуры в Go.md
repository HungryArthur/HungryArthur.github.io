В языке Go структуры позволяют моделировать сложные данные, комбинируя поля для представления сущностей. Для этого в Go существует семантика вложенных и встроенных структур, которые используются для создания иерархических данных, обеспечивая удобную организацию и доступ к данным. 

Рассмотрим как их правильно использовать и в чем они отличаются.

**Вложенная структура** — это структура, содержащая поле, типом которого является другая структура. Например, структура `Person` может включать поле `Addr` типа `Address`. Доступ к полям вложенной структуры осуществляется через точку, например, `person.Addr.City`. Вложенные структуры подходят для явной иерархии данных, где каждое поле имеет имя, что улучшает читаемость и чёткость структуры.

**Пример вложенной структуры:**
```go
package main

import (
	"errors"
	"fmt"
)

type Address struct {
	City  string
	Street string
}

func (a Address) GetFullAddress() string {
	return fmt.Sprintf("%s, %s", a.City, a.Street)
}

type Person struct {
	Name string
	Addr Address // Вложенная структура
}

func (p Person) Validate() error {
	if p.Name == "" || p.Addr.City == "" || p.Addr.Street == "" {
		return errors.New("invalid person data")
	}
	return nil
}

func (p *Person) ChangeAddress(newCity, newStreet string) error {
	if newCity == "" || newStreet == "" {
		return errors.New("invalid address data")
	}
	p.Addr.City = newCity
	p.Addr.Street = newStreet
	return nil
}

func main() {
	person := Person{
		Name: "Lena",
		Addr: Address{
			City:  "Moscow",
			Street: "Lenin St",
		},
	}
	err := person.Validate()
	if err != nil {
		fmt.Printf("Ошибка: %v\n", err)
		return
	}
	err = person.ChangeAddress("Moscow", "Park Druzhby")
	if err != nil {
		fmt.Printf("Ошибка: %v\n", err)
		return
	}
	fmt.Printf("Имя: %s, Полный адрес: %s\n", person.Name, person.Addr.GetFullAddress())
}
```
**Вывод:**
```makefile
Имя: Lena, Полный адрес: Moscow, Park Druzhby
```

**Пояснение:** Структура `Address` содержит поля `City` и `Street`. Структура `Person` включает поле `Addr` типа `Address`, что делает его вложенной структурой. Доступ к полям осуществляется через `person.Addr.City` и `person.Addr.Street`. Метод `GetFullAddress` возвращает форматированный адрес, вызывается как `person.Addr.GetFullAddress()`. Метод `Validate` проверяет, что все поля заполнены. Метод `ChangeAddress` с указательным ресивером изменяет поля `Addr` через `p.Addr.City = newCity` и `p.Addr.Street = newStreet`. В `main` создаётся `Person`, изменяется адрес и выводятся данные.

**Встроенная структура** — это структура, **включённая** в другую структуру без имени поля, только с указанием её типа (например, `Address` внутри `Person`). При необходимости, к этой встроенной структуре можно образаться по имени, равному имени типа. Это называется встраиванием (embedding).

Поля встроенной структуры доступны напрямую, например, `person.City` вместо `person.Address.City`. Встраивание упрощает синтаксис и позволяет наследовать поля и методы встроенной структуры.

**Пример встроенной структуры:**
```go
package main

import (
	"errors"
	"fmt"
)

type Address struct {
	City   string
	Street string
}

func (a Address) GetFullAddress() string {
	return fmt.Sprintf("%s, %s", a.City, a.Street)
}

type Person struct {
	Name    string
	Address // Встроенная структура
}

func (p Person) Validate() error {
	if p.Name == "" || p.City == "" || p.Street == "" {
		return errors.New("invalid person data")
	}
	return nil
}

func (p *Person) ChangeAddress(newCity, newStreet string) error {
	if newCity == "" || newStreet == "" {
		return errors.New("invalid address data")
	}
	p.City = newCity
	p.Street = newStreet
	return nil
}

func main() {
	person := Person{
		Name: "Ivan",
		Address: Address{
			City:   "London",
			Street: "Baker St",
		},
	}
	err := person.Validate()
	if err != nil {
		fmt.Printf("Ошибка: %v\n", err)
		return
	}
	err = person.ChangeAddress("Moscow", "Park Pokrovskoye-Streshnevo")
	if err != nil {
		fmt.Printf("Ошибка: %v\n", err)
		return
	}
	fmt.Printf("Имя: %s, Полный адрес: %s\n", person.Name, person.GetFullAddress())
}
```
**Вывод:**
```makefile
Имя: Ivan, Полный адрес: Moscow, Park Pokrovskoye-Streshnevo
```

**Пояснение:** Структура `Address` встроена в `Person` без имени поля, что позволяет обращаться к `City` и `Street` напрямую через `person.City` и `person.Street`. Метод `GetFullAddress` встроенной структуры продвигается и вызывается как `person.GetFullAddress()`. Метод `Validate` проверяет, что все поля заполнены. Метод `ChangeAddress` с указательным ресивером изменяет встроенные поля через `p.City = newCity` и `p.Street = newStreet`. В `main` создаётся `Person`, изменяется адрес и выводятся данные. Встраивание упрощает синтаксис, делая код более компактным.

Когда имя метода встроенной структуры совпадает с именем самой структуры, возникает **конфликт имён**. В Go это приводит к ошибке компиляции, так как имя типа встроенной структуры становится именем поля.

Чтобы разрешить конфликт, используйте квалифицированное имя для вызова метода встроенной структуры, например, `p.Address.Method()`, где `Address` — имя типа встроенной структуры.

**Пример с конфликтом имён:**
```go
package main

import (
	"errors"
	"fmt"
)

type Address struct {
	City  string
	Street string
}

func (a Address) Address() string { // Метод с именем, совпадающим с типом
	return fmt.Sprintf("%s, %s", a.City, a.Street)
}

type Person struct {
	Name string
	Address // Встроенная структура
}

func (p Person) Validate() error {
	if p.Name == "" || p.City == "" || p.Street == "" {
		return errors.New("invalid person data")
	}
	return nil
}

func (p *Person) ChangeAddress(newCity, newStreet string) error {
	if newCity == "" || newStreet == "" {
		return errors.New("invalid address data")
	}
	p.City = newCity
	p.Street = newStreet
	return nil
}

func main() {
	person := Person{
		Name: "Petr",
		Address: Address{
			City:  "New York",
			Street: "Broadway",
		},
	}
	err := person.Validate()
	if err != nil {
		fmt.Printf("Ошибка: %v\n", err)
		return
	}
	err = person.ChangeAddress("Moscow", "Molodezhnaya metro")
	if err != nil {
		fmt.Printf("Ошибка: %v\n", err)
		return
	}
	fmt.Printf("Имя: %s, Адрес: %s\n", person.Name, person.Address.Address()) // Квалифицированный вызов
}
```
**Вывод:**
```makefile
Имя: Petr, Адрес: Moscow, Molodezhnaya metro
```

**Пояснение:** Метод `Address` встроенной структуры имеет имя, совпадающее с типом `Address`. Для вызова метода используется квалифицированное имя `person.Address.Address()`, где первое `Address` — имя типа встроенной структуры, а второе — имя метода. Это разрешает конфликт и позволяет вызывать метод.
