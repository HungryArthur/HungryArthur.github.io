Структуры (struct) в Go — это пользовательский тип данных, который позволяет объединять несколько полей с разными типами в одну сущность. Они используются для моделирования объектов, таких как записи о людях, продуктах или других сущностях, где нужно хранить связанные данные вместе. Структуры обеспечивают удобный способ организации и работы с данными, представляя их в виде набора именованных полей.

**Что такое структура?**

Структура — это составной тип данных, определяемый с помощью ключевого слова `struct`. Она состоит из полей, каждое из которых имеет имя и тип. Структура объявляется с помощью синтаксиса `struct { ... }`. Например:

```haskell
type Person struct {
    Name string
    Age  int
}
```

Здесь `Person` — это тип структуры с двумя полями: `Name` типа `string` и `Age` типа `int`.

**Создание структур**

Структуры создаются следующими способами:

- Объявление переменной с инициализацией: `p := Person{Name: "Алекс", Age: 30}` — создаёт структуру с заданными значениями полей.
- Объявление без инициализации: `var p Person` — создаёт структуру с нулевыми значениями полей ("" для `string`, 0 для `int`).
- Частичная инициализация: `p := Person{Name: "Алекс"}` — указывается только поле `Name`, а `Age` получает нулевое значение (0).

После создания структуры её поля можно изменять или читать с помощью точечной нотации, например, `p.Name` или `p.Age = 31`.

**Свойства структур**

- Именованные поля: Каждое поле имеет имя и тип, что делает структуру читаемой и понятной.
- Фиксированный набор полей: Поля структуры задаются при её определении и не могут быть изменены.
- Нулевые значения: Неинициализированные поля получают значение по умолчанию для их типа (например, 0 для `int`, "" для `string`).

Размер структуры определяется количеством и типами её полей. Например, структура `Person` выше содержит строку и целое число.

**Операции со структурами**

- Инициализация: Поля можно задавать при создании структуры или изменять позже.
- Копирование: Присваивание структуры (например, `p2 := p1`) создаёт копию всех полей, так как структуры копируются по значению.
- Передача в функции: Структуры можно передавать в функции как аргументы, при этом передаётся копия структуры, также структуры можно передавать по ссылке, при этом изменять значения полей.

#### **Пример 1: Создание и изменение структуры**
```go
package main

import (
	"fmt"
)

type Person struct {
	Name string
	Age  int
}

func main() {
	p := Person{Name: "Алекс", Age: 25}
	fmt.Printf("Имя: %s, Возраст: %d\n", p.Name, p.Age)
	p.Age = 26
	fmt.Printf("После изменения: Имя: %s, Возраст: %d\n", p.Name, p.Age)
}
```
**Вывод:**
```makefile
Имя: Алекс, Возраст: 25
После изменения: Имя: Алекс, Возраст: 26
```

**Пояснение:** Создаётся структура `Person` с экспортируемыми полями `Name` и `Age`. Поле `Age` изменяется с 25 на 26 с помощью точечной нотации, так как оно экспортируемое.

####  **Пример 2: Передача структуры в функцию по копии**
```go
package main

import (
	"fmt"
)

type Point struct {
	X int
	Y int
}

func movePoint(p Point, dx int, dy int) {
	p.X += dx
	p.Y += dy
}

func main() {
	p := Point{X: 10, Y: 20}
	movePoint(p, 5, 3)
	fmt.Printf("Исходная точка после вызова функции с передачей аргумента по копии: X=%d, Y=%d\n", p.X, p.Y)
}
```
**Вывод:**
```1c
Исходная точка после вызова функции с передачей аргумента по копии: X=10, Y=20
```

**Пояснение:** Функция `movePoint` принимает структуру по копии `Point` с полями `X` и `Y`, изменяет их копию. Исходная структура остаётся неизменной, так как структуры копируются по значению.

####  **Пример 3: Передача структуры в функцию по ссылке**
```go
package main

import (
	"fmt"
)

type Point struct {
	X int
	Y int
}

func movePoint(p *Point, dx int, dy int) {
	p.X += dx
	p.Y += dy
}

func main() {
	p := Point{X: 10, Y: 20}
	movePoint(&p, 5, 3)
	fmt.Printf("Исходная точка: X=%d, Y=%d\n", p.X, p.Y)
}
```
**Вывод:**
```undefined
Исходная точка: X=15, Y=23
```

**Пояснение:** Функция `movePoint` принимает структуру `Point` по ссылке с полями `X` и `Y`, изменяет значения. Исходная структура при этом изменяема, так как поля структуры доступны по ссылке внутри функции.

**Рекомендации по использованию структур**
- Используйте структуры для группировки связанных данных, таких как характеристики объекта.
- Называйте поля с заглавной буквы для экспорта (доступа из других пакетов) или с маленькой для ограничения доступа внутри пакета.
- Инициализируйте только необходимые поля, если остальные могут иметь нулевые значения.
