В Go мапы (`map[K]V`) являются _ссылочным типом данных_. Мапа хранит указатель на базовую структуру данных (хеш-таблицу), а не сами данные. Когда мапа передаётся в функцию, передаётся копия этого указателя, а не копия всей мапы. Таким образом, изменения, сделанные в мапе внутри функции (например, добавление, удаление или обновление элементов), отражаются в оригинальной мапе, так как она ссылается на ту же базовую структуру.

**Пример передачи мапы в функцию:**
```go
package main

import "fmt"

func updateMap(m map[string]int, key string, value int) {
	m[key] = value // Изменяет оригинальную мапу
}

func main() {
	m := make(map[string]int)
	m["apple"] = 1
	fmt.Println("До:", m) // До: map[apple:1]
	updateMap(m, "banana", 2)
	fmt.Println("После:", m) // После: map[apple:1 banana:2]
}
```
**Вывод:**
```go
До: map[apple:1]
После: map[apple:1 banana:2]
```

**Пояснение:** В функции `updateMap` мапа `m` изменяется напрямую, так как она передаёт указатель на базовую структуру. Никакой явной передачи указателя (например, `*map[string]int`) не требуется.

#### Передача nil-мапы
Если передать в функцию `nil`-мапу, попытка записи в неё вызовет панику (`panic: assignment to entry in nil map`). Однако чтение из `nil`-мапы безопасно и возвращает нулевое значение типа значений мапы.

**Пример с nil-мапой:**
```go
package main

import "fmt"

func readMap(m map[string]int, key string) int {
	return m[key] // Безопасно, возвращает 0, если мапа nil
}

func main() {
	var m map[string]int // nil-мапа
	fmt.Println(readMap(m, "key")) // 0
	// m["key"] = 1 // Ошибка: panic: assignment to entry in nil map
}
```
**Вывод:**
```undefined
0
```

**Пояснение:** Перед использованием мапы для записи её нужно инициализировать с помощью `make` или литерала.

#### Мапы как значения мапы (вложенные мапы)

В Go значениями мапы могут быть другие мапы, создавая структуру вида `map[K1]map[K2]V`. Это полезно для представления сложных данных, например, группировки или иерархий. Однако перед использованием вложенной мапы её нужно инициализировать, иначе попытка записи вызовет панику.

**Пример вложенной мапы:**
```go
package main

import "fmt"

func main() {
	// Мапа, где значения — мапы
	nestedMap := make(map[string]map[string]int)
	
	// Инициализация вложенной мапы
	nestedMap["fruits"] = make(map[string]int)
	nestedMap["fruits"]["apple"] = 5
	nestedMap["fruits"]["banana"] = 3
	
	nestedMap["vegetables"] = make(map[string]int)
	nestedMap["vegetables"]["carrot"] = 2
	
	fmt.Println(nestedMap) // map[fruits:map[apple:5 banana:3] vegetables:map[carrot:2]]
}
```
 **Вывод:**
```go
map[fruits:map[apple:5 banana:3] vegetables:map[carrot:2]]
```

**Пояснение:** Перед добавлением пары в `nestedMap["fruits"]` необходимо инициализировать вложенную мапу с помощью `make`. Без этого возникнет паника.

#### Ключевые моменты
- **Ссылочный тип:** Мапы передаются по ссылке, изменения в функциях затрагивают оригинальную мапу.
- **Nil-мапы:** Чтение безопасно, запись требует инициализации.
- **Вложенные мапы:** Значениями мапы могут быть другие мапы, но вложенные мапы нужно инициализировать.
