**Интерфейс io.Reader**
Интерфейс `io.Reader` — один из самых распространённых в стандартной библиотеке Go. Он определён в пакете `io` и выглядит так:

```go
type Reader interface {
	Read(p []byte) (n int, err error)
}
```

`io.Reader` позволяет читать данные из различных источников, таких как файлы, сетевые соединения или строки, через единый контракт. Это делает код универсальным, так как функции могут работать с любым типом, реализующим `io.Reader`.

**Пример:**
Рассмотрим чтение данных из строки с использованием `io.Reader`.
```go
package main

import (
	"fmt"
	"strings"
)

type DataReader struct {
	Content string
}

func (dr *DataReader) Read(p []byte) (n int, err error) {
	reader := strings.NewReader(dr.Content)
	return reader.Read(p)
}

func processData(r io.Reader) string {
	data := make([]byte, 100)
	n, err := r.Read(data)
	if err != nil {
		return "Error: " + err.Error()
	}
	return string(data[:n])
}

func main() {
	defer fmt.Println("Reading completed")
	
	dr := DataReader{Content: "Sample data"}
	fmt.Println(processData(&dr))
}
```
**Вывод:**
```haskell
Sample data
Reading completed
```

**Пояснение:**
`DataReader` реализует `io.Reader`, используя `strings.NewReader` для чтения строки. Функция `processData` принимает любой `io.Reader`, что позволяет ей работать с разными источниками данных. `defer` выводит `"Reading completed"`.

#### Интерфейс io.Writer
Интерфейс `io.Writer` также определён в пакете `io` и выглядит так:
```go
type Writer interface {
	Write(p []byte) (n int, err error)
}
```

`io.Writer` используется для записи данных в различные цели, такие как консоль, файлы или буферы. Он часто применяется в связке с `io.Reader` для обработки потоков данных.

**Пример:**
Рассмотрим запись данных в буфер с использованием `io.Writer`.
```go
package main

import (
	"bytes"
	"fmt"
	"io"
)

// DataWriter реализует интерфейс io.Writer
type DataWriter struct {
	Content string
}

// Write реализует метод записи в DataWriter
func (dw *DataWriter) Write(p []byte) (n int, err error) {
	dw.Content = string(p)
	return len(p), nil
}

// writeData записывает строку в любой io.Writer
func writeData(w io.Writer, content string) error {
	_, err := fmt.Fprint(w, content)
	return err
}

func main() {
	// Пример 1: Запись в bytes.Buffer
	var buffer bytes.Buffer
	err := writeData(&buffer, "Report data")
	if err != nil {
		fmt.Println("Error:", err)
		return
	}
	fmt.Println("Buffer:", buffer.String())
	
	// Пример 2: Запись в DataWriter
	var dataWriter DataWriter
	err = writeData(&dataWriter, "Custom data")
	if err != nil {
		fmt.Println("Error:", err)
		return
	}
	fmt.Println("DataWriter:", dataWriter.Content)
	
	defer fmt.Println("Writing completed")
}
```
**Вывод:**
```haskell
Buffer: Report data
DataWriter: Custom data
Writing completed
```

**Пояснение:**
`DataWriter` реализует `io.Writer`, хотя в примере метод упрощён. Функция `writeData` записывает данные в любой `io.Writer`, в данном случае в `bytes.Buffer`и в `DataWriter`. `defer` выводит `"Writing completed"`.

#### Интерфейс Stringer
Интерфейс `Stringer` определён в пакете `fmt` и выглядит так:
```typescript
type Stringer interface {
	String() string
}
```

Типы, реализующие `Stringer`, могут определять, как они будут представлены в виде строки, например, при использовании `fmt.Println`. Это полезно для форматирования вывода пользовательских типов.

**Пример:**
Рассмотрим форматирование структуры для отображения информации о продукте.
```go
package main

import "fmt"

type Product struct {
	Name  string
	Price int
}

func (p Product) String() string {
	return fmt.Sprintf("Product: %s, Price: %d", p.Name, p.Price)
}

func main() {
	defer fmt.Println("Formatting completed")
	
	product := Product{Name: "Book", Price: 25}
	fmt.Println(product)
}
```
**Вывод:**
```yaml
Product: Book, Price: 25
Formatting completed
```

**Пояснение:**
`Product` реализует `Stringer` через метод `String`, определяя, как структура отображается при печати. `fmt.Println` автоматически вызывает `String`. `defer` выводит `"Formatting completed"`.
