**Массивы в Go** — это структуры данных, предназначенные для хранения фиксированного количества элементов одного типа в упорядоченной последовательности. Они подходят для задач, где число элементов заранее известно, например, для учёта количества проданных билетов на выставку за три дня. Длина массива задаётся при его создании и не может измениться, что делает массивы предсказуемыми и эффективными. В Go длина массива является частью его типа, поэтому массив из трёх целых чисел отличается от массива из четырёх целых чисел. Это обеспечивает строгую типизацию, но требует заранее знать, сколько элементов нужно хранить.

**Объявление массива** выполняется с указанием типа элементов и фиксированной длины. Например, можно создать массив для хранения количества билетов, проданных в разные дни выставки. При объявлении массива без явной инициализации Go автоматически заполняет его элементы значениями по умолчанию для соответствующего типа. Для целых чисел это нули, для строк — пустые строки. Такое поведение гарантирует, что массив всегда имеет определённое состояние, но значения по умолчанию могут быть неподходящими для реальных данных. Например, нули в массиве билетов могут быть восприняты как отсутствие продаж, что может не соответствовать действительности.

**Пример 1:**
```go
package main

import "fmt"

func main() {
	var ticketSales [3]int
	fmt.Println("Ticket sales:", ticketSales)
}
```

**Вывод:**
```less
Ticket sales: [0 0 0]
```

**Пояснение:** Массив ticketSales объявлен без инициализации, поэтому содержит нули. Если программа подсчитывает реальные продажи билетов, нулевые значения могут быть ошибочно приняты за отсутствие продаж, что подчёркивает важность инициализации.

**Инициализация массива** позволяет задать конкретные значения, чтобы они отражали реальные данные. В Go массив можно инициализировать при объявлении, указав значения в фигурных скобках. Это удобно, когда начальные данные известны, например, когда есть статистика продаж билетов за три дня выставки. Инициализация при объявлении делает код понятным, сразу показывая содержимое массива, и предотвращает использование нулевых значений, которые могут привести к ошибкам в вычислениях, таких как подсчёт общего числа проданных билетов. Число аргументов при этом должно быть равно размеру массива.

**Пример 2:**
```go
package main

import "fmt"

func main() {
	ticketCounts := [3]int{200, 250, 180}
	fmt.Println("Ticket counts:", ticketCounts)
}
```

**Вывод:**

```less
Ticket counts: [200 250 180]
```

**Пояснение:** Массив ticketCounts инициализирован значениями, представляющими количество проданных билетов за три дня. Это исключает проблему нулевых значений и делает данные готовыми для использования, например, для вычисления общей выручки.

**Инициализация по индексам** — ещё один способ задать значения массива после его объявления. Этот подход полезен, когда данные поступают постепенно, например, когда статистика продаж билетов обновляется по мере поступления отчётов. Элементы массива доступны по индексам, которые начинаются с нуля и заканчиваются длиной массива минус один. Инициализация по индексам позволяет обновлять только те элементы, для которых есть данные, но требует аккуратности, чтобы не пропустить важные значения.

**Пример 3:**

```go
package main

import "fmt"

func main() {
	var dailyTickets [3]int
	dailyTickets[0] = 190 // Первый день
	dailyTickets[2] = 210 // Третий день
	fmt.Println("Daily tickets:", dailyTickets)
}
```

**Вывод:**

```less
Daily tickets: [190 0 210]
```

**Пояснение:** Массив dailyTickets инициализирован по индексам для первого и третьего дня, а второй день остался нулём. Это показывает гибкость инициализации по индексам, но также подчёркивает, что неинициализированные элементы содержат нули, которые могут быть некорректными в контексте задачи.

**Чтение элементов массива**

Чтение элементов массива выполняется с использованием индекса, указанного в квадратных скобках. Например, чтобы получить количество билетов, проданных во второй день, можно обратиться к элементу массива по индексу 1. Это делает массивы удобными для быстрого доступа к данным, но требует, чтобы индекс был в пределах допустимого диапазона. Считывание позволяет использовать данные массива для вычислений, таких как подсчёт среднего числа проданных билетов или определение самого активного дня.

**Пример 4:**

```go
package main

import "fmt"

func main() {
	exhibitionTickets := [3]int{220, 270, 195}
	secondDay := exhibitionTickets[1]
	fmt.Println("Second day tickets:", secondDay)
}
```

**Вывод:**

```sql
Second day tickets: 270
```

**Пояснение:** Массив exhibitionTickets инициализирован, и программа считывает значение по индексу 1, возвращая количество билетов за второй день. Инициализация массива обеспечивает корректные данные для считывания.

Ссылка на песочницу: [https://go.dev/play/p/EBVXrQjE-Q2](https://go.dev/play/p/EBVXrQjE-Q2)

**Последствия отсутствия инициализации**

Если массив не инициализировать, он заполняется значениями по умолчанию, которые могут не соответствовать реальной задаче. Например, в массиве для учёта книг нули могут быть интерпретированы как отсутствие книг в категории, тогда как на самом деле данные могли ещё не поступить или количество книг ненулевое. Это может привести к ошибкам в вычислениях, например, к неверному подсчёту общего числа книг. Инициализация позволяет задать осмысленные значения, делая программу надёжной и понятной.

**Пример 5:**

```go
package main

import "fmt"

func main() {
	var libraryStock [3]int
	total := libraryStock[0] + libraryStock[1] + libraryStock[2]
	fmt.Println("Total books:", total)
}
```

**Вывод:**

```yaml
Total books: 0
```

**Пояснение:** Без инициализации массив libraryStock содержит нули, и сумма также равна нулю, что не отражает реального количества книг. Инициализация могла бы задать реальные значения, например, {320, 170, 210}, чтобы получить корректную сумму.

**Обращение по неверному индексу** — это попытка доступа к элементу массива за пределами его длины, например, по индексу меньше нуля или больше либо равен длине массива. В Go такое обращение вызывает ошибку во время компиляции программы, поскольку уже в этот момент известно, что элемента с таким индексом быть не может, так как размер массива, как мы уже знаем, задается при создании и является неизменяемым типом.

Например, если массив имеет длину 3, попытка получить элемент по индексу 3 или -1 вызовет панику. Это делает важной проверку индексов перед обращением, чтобы избежать ошибок, особенно в программах, где индексы зависят от пользовательского ввода.

**Пример 6:**

```go
package main

import "fmt"

func main() {
	ticketRecords := [3]int{180, 230, 160}
	fmt.Println("Invalid index access:", ticketRecords[3])
}
```

**Вывод:**

```delphi
./prog.go:7:53: invalid argument: index 3 out of bounds [0:3]
```

**Пояснение:** Массив ticketRecords имеет длину 3, поэтому индексы 0, 1 и 2 валидны. Обращение к ticketRecords[3] вызывает ошибку во время компиляции, так как индекс выходит за пределы массива. Это подчёркивает необходимость проверки индексов в программе.

Ссылка на песочницу: [https://go.dev/play/p/Cu-mkp47dMP](https://go.dev/play/p/Cu-mkp47dMP)

**Ограничение на динамическую длину**

В Go длина массива должна быть известна на этапе компиляции, поэтому нельзя создать массив с динамической длиной, определяемой во время выполнения, например, через ввод пользователя с помощью `fmt.Scan`. Попытка использовать переменную для задания длины массива приведёт к ошибке компиляции, так как Go требует, чтобы размер массива был константой или литералом. Это ограничение связано с тем, что массивы в Go имеют фиксированный размер, являющийся частью их типа. 

**Пример 7:**

```go
package main

import "fmt"

func main() {
	var n int
	fmt.Scan(&n)
	var dynamicArray [n]int
	fmt.Println("Dynamic array:", dynamicArray)
}
```

**Вывод:**

```bash
./prog.go:7:14: invalid array length n
```

**Пояснение:** Попытка создать массив dynamicArray с длиной n, введённой через `fmt.Scan`, приводит к ошибке компиляции, так как длина массива должна быть константой, известной на этапе компиляции. Например, нужно использовать `var dynamicArray [3]int` с фиксированной длиной 3.
