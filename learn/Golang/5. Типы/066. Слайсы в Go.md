В предыдущей главе мы изучили массивы — структуры данных с фиксированным размером, заданным на этапе компиляции. Массивы удобны для работы с заранее известным количеством элементов, но их неизменяемый размер ограничивает гибкость. В этой главе мы разберём так называемые динамические массивы, или **слайсы**, которые представляют собой более гибкую альтернативу массивам в Go. Слайсы позволяют динамически изменять размер, добавлять или удалять элементы, сохраняя при этом удобство работы с последовательностями данных. 

#### Основы слайсов
Слайс в Go — это структура данных, которая предоставляет доступ к подмножеству элементов базового массива. Слайс не хранит данные самостоятельно, а является "адаптером" над массивом, определяемым тремя компонентами:
- **Указатель:** Ссылка на начало подмножества в базовом массиве.
- **Длина (length):** Количество элементов, доступных в слайсе (`len(slice)`).
- **Ёмкость (capacity):** Максимальное количество элементов от начала слайса до конца базового массива (`cap(slice)`).

Синтаксис объявления слайса: `[]T`, где `T` — тип элементов, например, `[]int`. В отличие от массивов, слайсы не требуют указания размера при объявлении.

#### Инициализация слайсов
Слайсы можно инициализировать несколькими способами:
- **С помощью литерала:** `s := []int{1, 2, 3}` создаёт слайс с длиной и ёмкостью 3, опирающийся на базовый массив, содержащий элементы 1, 2, 3.
- **С помощью `make`:** Функция `make` позволяет создать слайс с заданной длиной и, опционально, ёмкостью. Синтаксис:
    - `make([]T, len)` — создаёт слайс с длиной и ёмкостью, равными `len`
    - `make([]T, len, cap)` — создаёт слайс с длиной `len` и ёмкостью `cap`, где `cap >= len`
    Все элементы инициализируются нулевым значением для типа `T` (например, 0 для `int`)
- **Из массива:** `arr := [5]int{1, 2, 3, 4, 5}; s := arr[1:4]` создаёт слайс, включающий элементы массива с индексов 1 по 3 (длина 3, ёмкость 4, так как слайс начинается с индекса 1).

**Пример инициализации слайса:**
```go
package main

import "fmt"

func main() {
	// Инициализация с помощью литерала
	s1 := []int{1, 2, 3}
	fmt.Printf("Слайс s1: %v, длина: %d, ёмкость: %d\n", s1, len(s1), cap(s1))
	
	// Инициализация с помощью make
	s2 := make([]int, 3, 5)
	fmt.Printf("Слайс s2: %v, длина: %d, ёмкость: %d\n", s2, len(s2), cap(s2))
	
	// Инициализация из массива
	arr := [5]int{10, 20, 30, 40, 50}
	s3 := arr[1:4]
	fmt.Printf("Слайс s3: %v, длина: %d, ёмкость: %d\n", s3, len(s3), cap(s3))
}
```
**Вывод:**
```yaml
Слайс s1: [1 2 3], длина: 3, ёмкость: 3
Слайс s2: [0 0 0], длина: 3, ёмкость: 5
Слайс s3: [20 30 40], длина: 3, ёмкость: 4
```

**Пояснение:** `s1` инициализируется с элементами 1, 2, 3; `s2` создаётся с длиной 3 и ёмкостью 5, заполненной нулями; `s3` берёт элементы 20, 30, 40 из массива, с ёмкостью 4, так как начинается с индекса 1.

Создание слайса с предварительно заданной емкостью (capacity) позволяет выделить необходимый объем памяти заранее. Это предотвращает многократные переаллокации массива при добавлении новых элементов, что значительно повышает производительность программы, особенно при работе с большими объемами данных.

#### Итерация по слайсам
Итерация по слайсам выполняется с помощью цикла `for`, либо в виде обычного цикла с индексами, либо с использованием конструкции `range`. Цикл `range` возвращает индекс и значение элемента, что удобно для обработки элементов слайса. Если индекс не нужен, его можно опустить, используя `_`.

Слайсы часто заполняются данными, считанными из ввода. В Go для чтения чисел можно использовать `fmt.Scan`, который позволяет напрямую считывать целые числа в элементы слайса.

**Пример итерации и считывания ввода:**
```go
package main

import "fmt"

func main() {
	var n int
	fmt.Scan(&n) // Считываем количество элементов
	s := make([]int, n) // Создаём слайс с длиной n
	for i := 0; i < n; i++ {
		fmt.Scan(&s[i]) // Считываем элементы
	}
	sum := 0
	for i, val := range s { // Итерация с помощью range
		fmt.Printf("Элемент %d: %d\n", i, val)
		sum += val
	}
	fmt.Printf("Сумма элементов: %d\n", sum)
}
```
**Ввод:**
```undefined
4
1 2 3 4
```
**Вывод:**
```yaml
Элемент 0: 1
Элемент 1: 2
Элемент 2: 3
Элемент 3: 4
Сумма элементов: 10
```

**Пояснение:** Программа считывает количество элементов `n`, создаёт слайс с помощью `make([]int, n)`, заполняет его `n` числами с помощью `fmt.Scan`, затем итерируется по слайсу с помощью `range`, выводя индекс и значение каждого элемента, а также вычисляет их сумму.

#### Что будет, если не инициализировать слайс?
Если слайс объявлен, но не инициализирован, он имеет значение `nil`. Нулевой слайс (`var s []int`) имеет следующие характеристики:
- **Длина:** 0 (`len(s) == 0`).
- **Ёмкость:** 0 (`cap(s) == 0`).
- **Указатель:** Не указывает на базовый массив (равен `nil`).

При попытке доступа к элементам нулевого слайса (например, `s[0]`) программа завершится с ошибкой `panic: runtime error: index out of range`. 

**Пример с несуществующим индексом:**
```go
package main

import "fmt"

func main() {
	var s []int // Нулевой слайс
	fmt.Print(s[6])
}
```
**Вывод:**
```applescript
panic: runtime error: index out of range [6] with length 0

goroutine 1 [running]:
main.main()
	/tmp/sandbox433503313/prog.go:7 +0x14
```

**Пояснение:** Нулевой слайс `s` имеет длину и ёмкость 0 и является `nil`. При доступе к несуществующему элементу по шестому индексу возникает паника

Однако нулевой слайс можно использовать с `append`, так как Go автоматически создаст базовый массив.

**Пример с нулевым слайсом:**
```go
package main

import "fmt"

func main() {
	var s []int // Нулевой слайс
	fmt.Printf("Нулевой слайс: %v, длина: %d, ёмкость: %d, nil: %t\n", s, len(s), cap(s), s == nil)
	s = append(s, 42)
	fmt.Printf("После append: %v, длина: %d, ёмкость: %d, nil: %t\n", s, len(s), cap(s), s == nil)
}
```
**Вывод:**
```go
Нулевой слайс: [], длина: 0, ёмкость: 0, nil: true
После append: [42], длина: 1, ёмкость: 1, nil: false
```

**Пояснение:** Нулевой слайс `s` имеет длину и ёмкость 0 и является `nil`. После `append` он становится ненулевым слайсом с базовым массивом.
