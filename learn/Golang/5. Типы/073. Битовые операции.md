Побитовые операции в Go манипулируют битами целых чисел, выполняя действия на каждом бите отдельно. В этом шаге будут приведены пошаговые разборы операций с использованием примеров. 

#### 1. Побитовое И (`&`)
**Описание:** Возвращает `1` для позиций, где оба бита равны `1`, иначе `0`.

**Пример: `5 & 3`**
- **Шаг 1:** Представим числа в двоичной системе (для простоты используем 4 бита):  
    `5 = 0101`  
    `3 = 0011`
- **Шаг 2:** Выполняем побитовое И для каждой позиции:  
    `0 & 0 = 0` (позиция 3)  
    `1 & 0 = 0` (позиция 2)  
    `0 & 1 = 0` (позиция 1)  
    `1 & 1 = 1` (позиция 0)
- **Шаг 3:** Получаем результат: `0001`.
- **Шаг 4:** Преобразуем в десятичную систему: `0001 = 1`.
- **Результат:** `5 & 3 = 1`.

#### 2. Побитовое ИЛИ (`|`)
**Описание:** Возвращает `1`, если хотя бы один бит равен `1`, иначе `0`.

**Пример: `5 | 3`**
- **Шаг 1:** Двоичное представление:  
    `5 = 0101`  
    `3 = 0011`
- **Шаг 2:** Выполняем побитовое ИЛИ:  
    `0 | 0 = 0` (позиция 3)  
    `1 | 0 = 1` (позиция 2)  
    `0 | 1 = 1` (позиция 1)  
    `1 | 1 = 1` (позиция 0)
- **Шаг 3:** Получаем результат: `0111`.
- **Шаг 4:** Преобразуем: `0111 = 4 + 2 + 1 = 7`.
- **Результат:** `5 | 3 = 7`.

#### 3. Побитовое исключающее ИЛИ (XOR, `^`)
**Описание:** Возвращает `1`, если биты различаются, иначе `0`.

**Пример: `5 ^ 3`**
- **Шаг 1:** Двоичное представление:  
    `5 = 0101`  
    `3 = 0011`
- **Шаг 2:** Выполняем XOR:  
    `0 ^ 0 = 0` (позиция 3)  
    `1 ^ 0 = 1` (позиция 2)  
    `0 ^ 1 = 1` (позиция 1)  
    `1 ^ 1 = 0` (позиция 0)
- **Шаг 3:** Получаем результат: `0110`.
- **Шаг 4:** Преобразуем: `0110 = 4 + 2 = 6`.
- **Результат:** `5 ^ 3 = 6`.

#### 4. Побитовое НЕ (`^`)
**Описание:** Инвертирует все биты числа. Для знаковых типов (например, `int32`) учитывает знак, так как инверсия влияет на весь регистр, включая старшие биты.

**Пример: `^5` для `int32`**
- **Шаг 1:** Двоичное представление для `int32`:  
    `5 = 00000000...00000101` (32 бита).
- **Шаг 2:** Инвертируем биты:  
    `^00000000...00000101 = 11111111...11111010`.
- **Шаг 3:** Для `int32` это отрицательное число (знаковый бит `1`). Преобразуем в десятичное: `111...11010 = -6` (в дополнительном коде: инверсия `5` даёт `-5-1 = -6`).
- **Результат:** `^5 = -6` (для `int32`).

#### 5. Сдвиг влево (`<<`)
**Описание:** Сдвигает биты влево, добавляя нули справа. Эквивалентно умножению на `2^k`, где `k` — величина сдвига.

**Пример: `5 << 1`**
- **Шаг 1:** Двоичное представление:  
    `5 = 0101`.
- **Шаг 2:** Сдвигаем на 1 бит влево:  
    `0101 << 1 = 1010`.
- **Шаг 3:** Преобразуем: `1010 = 8 + 2 = 10`.
- **Результат:** `5 << 1 = 10`.

#### 6. Сдвиг вправо (`>>`)
**Описание:** Сдвигает биты вправо, заполняя старшие биты нулём (для беззнаковых типов) или битом знака (для знаковых). Эквивалентно делению на `2^k` с округлением вниз.

**Пример: `5 >> 1`**
- **Шаг 1:** Двоичное представление:  
    `5 = 0101`.
- **Шаг 2:** Сдвигаем на 1 бит вправо (знаковый бит `0`):  
    `0101 >> 1 = 0010`.
- **Шаг 3:** Преобразуем: `0010 = 2`.
- **Результат:** `5 >> 1 = 2`.

**Ключевые моменты:**
- Побитовые операции работают с целыми числами (`int`, `uint`, `int32`, и т.д.). Для знаковых типов побитовое НЕ и сдвиги учитывают знак.
- Для бинарных строк операции требуют посимвольной обработки или конвертации в числа (например, через `strconv.ParseInt`).
- Сдвиги могут привести к переполнению для больших чисел, особенно в знаковых типах, с этим необходимо быть крайне аккуратным.
