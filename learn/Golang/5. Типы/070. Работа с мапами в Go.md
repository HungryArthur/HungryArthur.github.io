Мапы (maps) в Go — это встроенная структура данных, представляющая ассоциативный массив (словарь), где каждому ключу соответствует значение. Мапы обеспечивают быстрый доступ, вставку и удаление элементов по ключу. Они идеально подходят для хранения пар ключ-значение, например, для подсчёта частоты или создания словарей.

#### Создание и инициализация
Мапы в Go должны быть инициализированы перед использованием, иначе они имеют значение `nil`, и попытка записи в `nil`-мапу вызовет панику (`panic: assignment to entry in nil map`). Инициализация выполняется с помощью `make` или литерала мапы.

**Инициализация с `make` и указанием ёмкости:**
```go
package main

import "fmt"

func main() {
	m := make(map[string]int, 10) // Инициализация с начальной ёмкостью 10
	m["apple"] = 5
	m["banana"] = 3
	fmt.Println(m)
	fmt.Println(len(m))
}
```
 **Вывод:**
```css
map[apple:5 banana:3]
2
```

**Пояснение:** `make(map[K]V, capacity)` создаёт пустую мапу с типом ключа `K` и значений `V`, где `capacity` — начальная ёмкость, которая определяет количество элементов, которое мапа может хранить без перераспределения памяти. Это может улучшить производительность при добавлении большого числа элементов, но не ограничивает рост мапы (она автоматически расширяется при необходимости). Но можно не указывать сapacity и инициализировать следующим образом: `make(map[K]V)`, тогда `capacity` будет равно 0.

**Инициализация с литералом:**
```go
package main

import "fmt"

func main() {
	m := map[string]int{
		"apple":  5,
		"banana": 3,
	}
	fmt.Println(m)
}
```
 **Вывод:**
```css
map[apple:5 banana:3]
```

**Пояснение:** Литерал `map[K]V{key: value}` позволяет сразу задать пары ключ-значение. Без инициализации (`var m map[string]int`) мапа будет `nil`, и запись в неё вызовет панику.

#### Операции с мапами
- **Добавление/обновление:** `m[key] = value`. Если элемент `key` уже есть в мапе, то его значение **перезапишется**.
- **Чтение:** `value := m[key]`. Если ключа нет, возвращается нулевое значение типа значения (например, `0` для `int`).
- **Проверка наличия ключа:** `value, ok := m[key]`. `ok` — `true`, если ключ существует, иначе `false`.
- **Удаление:** `delete(m, key)` удаляет ключ из мапы.
- **Итерация:** Используется `for key, value := range m`. Порядок итерации случайный и не гарантирован.

**Пример операций:**
```go
package main

import "fmt"

func main() {
	m := make(map[string]int, 5) // Ёмкость 5
	m["apple"] = 5
	m["banana"] = 3
	fmt.Println(m["apple"]) // 5
	if val, ok := m["orange"]; ok {
		fmt.Println(val)
	} else {
		fmt.Println("orange not found") // orange not found
	}
	delete(m, "banana")
	for key, value := range m {
		fmt.Printf("%s: %d\n", key, value) // apple: 5 (порядок не гарантирован)
	}
}
```
 **Вывод:**
```makefile
5
orange not found
apple: 5
```

**Пояснение:** Мапа инициализируется с ёмкостью 5, добавляются пары, проверяется наличие ключа, удаляется пара, выполняется итерация. Порядок вывода ключей при использовании `range` случайный, так как Go намеренно рандомизирует порядок итерации по мапе для предотвращения зависимости от внутренней реализации.

#### Итерация с помощью `for ... range`
Для обхода мапы используется конструкция `for key, value := range m`. Если нужно только ключи, можно использовать `for key := range m`. Порядок обхода не определён, и каждая итерация может возвращать ключи в разном порядке.

**Пример итерации:**
```go
package main

import "fmt"

func main() {
	m := map[string]int{"a": 1, "b": 2, "c": 3}
	for k, v := range m {
		fmt.Printf("%s: %d\n", k, v) // Порядок вывода случайный, например, b:2, c:3, a:1
	}
}
```
 **Вывод:**
```makefile
a: 1
b: 2
c: 3
```

**Пояснение:** Порядок вывода пар ключ-значение не фиксирован. Если требуется определённый порядок, нужно собрать ключи в слайс и отсортировать его (например, с помощью `slices.Sort`).

#### Типы ключей: Comparable
Ключи мапы должны быть _comparable_, то есть поддерживать операцию сравнения (`==` и `!=`)
К comparable типам относятся:
- Числовые типы (`int`, `float64`, etc.).
- Строки (`string`).
- Булевы значения (`bool`).
- Указатели.
- Интерфейсы, если их конкретные значения comparable.
- Структуры и массивы, если все их поля/элементы comparable.

Типы, такие как слайсы, мапы и функции, не являются comparable, поэтому их нельзя использовать в качестве ключей мапы. Попытка это сделать приведёт к ошибке компиляции.

**Пример недопустимого ключа:**
```go
package main

func main() {
	var m map[[]int]int // Ошибка компиляции: invalid map key type []int
}
```
**Вывод:**
```go
./prog.go:4:6: declared and not used: m
./prog.go:4:12: invalid map key type []int
```

#### Особенности
- **Случайный порядок итерации:** Go рандомизирует порядок ключей при обходе мапы, чтобы код не зависел от внутренней реализации.
- **Ёмкость:** Указание ёмкости в `make` оптимизирует выделение памяти, но мапа автоматически расширяется при необходимости.
- **Длина:** `len(m)` возвращает количество пар ключ-значение.
