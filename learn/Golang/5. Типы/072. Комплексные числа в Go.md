[Комплексные числа](https://stepik.org/lesson/2001714/step/%C2%A0https://ru.wikipedia.org/wiki/%D0%9A%D0%BE%D0%BC%D0%BF%D0%BB%D0%B5%D0%BA%D1%81%D0%BD%D0%BE%D0%B5_%D1%87%D0%B8%D1%81%D0%BB%D0%BE) в Go — это встроенные типы данных `complex64` и `complex128`, представляющие числа вида `a + bi`, где `a` — действительная часть, `b` — мнимая часть, а `i` — мнимая единица (`i² = -1`). Тип `complex64` использует два `float32` для хранения действительной и мнимой частей, а `complex128` — два `float64`, обеспечивая большую точность.

**Создание и синтаксис:** Комплексное число создаётся с помощью функции `complex(real, imag)`, где `real` и `imag` — числа типа `float32` (для `complex64`) или `float64` (для `complex128`). Например, `z := complex(2.0, 3.0)` создаёт `2 + 3i` типа `complex128`. Если аргументы — `float64`, результат — `complex128`; если `float32` — `complex64`.

**Операции:** Go поддерживает стандартные арифметические операции для комплексных чисел:
- **Сложение:** `(a + bi) + (c + di) = (a + c) + (b + d)i`.
- **Вычитание:** `(a + bi) - (c + di) = (a - c) + (b - d)i`.
- **Умножение:** `(a + bi)(c + di) = (ac - bd) + (ad + bc)i`.
- **Деление:** `(a + bi)/(c + di) = ((ac + bd)/(c² + d²)) + ((bc - ad)/(c² + d²))i`.

**Встроенные функции:**
- `real(z)`: возвращает действительную часть числа `z`.
- `imag(z)`: возвращает мнимую часть числа `z`.

**Пакет `math/cmplx`:** Дополнительные операции доступны через пакет `math/cmplx`:
- `cmplx.Abs(z)`: модуль числа `z = a + bi`, равный `√(a² + b²)`.
- `cmplx.Conj(z)`: комплексно-сопряжённое число `a - bi`.
- `cmplx.Phase(z)`: аргумент (угол) числа `z` в радианах.
- `cmplx.Sqrt(z)`, `cmplx.Pow(z, n)`: квадратный корень и возведение в степень.

**Пример:**
```go
package main

import (
	"fmt"
	"math/cmplx"
)

func main() {
	z1 := complex(3.0, 4.0) // 3 + 4i
	z2 := complex(1.0, 2.0) // 1 + 2i
	sum := z1 + z2
	prod := z1 * z2
	fmt.Printf("Сумма: %.1f + %.1fi\n", real(sum), imag(sum)) // Сумма: 4.0 + 6.0i
	fmt.Printf("Произведение: %.1f + %.1fi\n", real(prod), imag(prod)) // Произведение: -5.0 + 10.0i
	fmt.Printf("Модуль z1: %.1f\n", cmplx.Abs(z1)) // Модуль z1: 5.0
}
```
**Вывод:**
```makefile
Сумма: 4.0 + 6.0i
Произведение: -5.0 + 10.0i
Модуль z1: 5.0
```

**Ключевые моменты:**
- Комплексные числа — это значения, а не ссылочный тип. Передача в функции копирует значение, но функции могут возвращать изменённые результаты.
- Для высокой точности используйте `complex128`, так как `complex64` может терять точность при сложных вычислениях.
- Пакет `math/cmplx` необходим для операций, выходящих за рамки базовой арифметики.
