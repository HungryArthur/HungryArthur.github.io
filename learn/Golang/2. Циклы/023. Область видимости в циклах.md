Циклы `for` создают свою область видимости для переменных, объявленных в инициализации цикла. Переменные, объявленные в `for init; condition; post`, доступны только внутри тела цикла. Переменные, объявленные в теле цикла, существуют только в пределах итерации или блока. Это помогает изолировать счётчики цикла и избежать конфликтов. Если переменная объявлена вне цикла, она доступна внутри, но изменения в цикле сохраняются после выхода. Аналогично, вложенные циклы имеют свои области, где внутренний цикл видит внешние переменные, но не наоборот.

**Пример:**
```go
package main

import "fmt"

func main() {
	var sum int
	for i := 1; i <= 5; i++ { // i доступна только в цикле
		sum += i
	}
	// fmt.Println(i) // Ошибка: i не определена
	message := fmt.Sprintf("Сумма: %d", sum)
	fmt.Println(message)
}
```
**Вывод:**
```makefile
Сумма: 15
```

**Комментарий:** Переменная `i` существует только в цикле `for`. Попытка использовать `i` снаружи вызовет ошибку компиляции. Внешняя `sum` доступна и изменяется в цикле.

#### Область видимости и операто defer

Рассмотрим оператор `defer`, который позволяет откладывать выполнение действия до выхода из функции. Он работает с переменными из своей области видимости на момент объявления. Если переменная затеняется в блоке, `defer` использует значение внешней переменной. `defer` выполняется в обратном порядке (LIFO), если их несколько, и всегда выполняется, даже при ошибках. Это полезно для очистки ресурсов или отложенного вывода, но требует внимания к области видимости, чтобы переменные были доступны на момент выполнения `defer`.

**Пример:**
```go
package main

import "fmt"

func main() {
	result := 5
	defer fmt.Println("Завершено с result =", result)
	{
		result := 15 // Затеняет внешнюю result
		message := fmt.Sprintf("Внутри блока: result = %d", result)
		fmt.Println(message)
	}
	message := fmt.Sprintf("Снаружи блока: result = %d", result)
	fmt.Println(message)
}
```
**Вывод:**
```sql
Внутри блока: result = 15
Снаружи блока: result = 5
Завершено с result = 5
```

**Комментарий:** `defer` использует внешнюю `result` (5), так как она была определена до затенения.

#### Управление видимостью defer

Оператор `defer` захватывает переменные из своей области видимости на момент объявления, но выполняет их с текущими значениями на момент выхода. Если переменная изменяется после `defer`, отложенное действие увидит новое значение. При затенении `defer` использует переменную из своей области. 

**Пример:**
```go
package main

import "fmt"

func main() {
	result := 5
	defer fmt.Println("Завершено с result =", result)
	result = 10
	defer fmt.Println("Второе defer: result =", result)
	fmt.Println("Середина: result =", result)
}
```
**Вывод:**
```sql
Середина: result = 10
Второе defer: result = 10
Завершено с result = 5
```

**Комментарий:** `defer` выполняются в обратном порядке: сначала последнее, затем первое. Изменение `result` после первого `defer` влияет на оба, так как они захватывают `result` по ссылке, подробнее о том, что это, будет расказано далее в курсе, и видят её текущее значение на момент выхода из функции.
