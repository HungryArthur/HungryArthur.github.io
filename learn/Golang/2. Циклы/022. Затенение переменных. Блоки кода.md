#### Затенение переменных

Затенение (shadowing) происходит, когда переменная во вложенном блоке имеет то же имя, что и внешняя переменная. Внутренняя переменная скрывает внешнюю в своей области видимости, но не изменяет её значения. После выхода из блока внешняя переменная снова становится доступной. Затенение может быть полезным для локальной переопределения поведения, но часто приводит к ошибкам, если программист случайно использует внутреннюю переменную вместо внешней. Это особенно актуально при использовании `:=` в блоках, где новая переменная затеняет внешнюю, что может привести к неожиданному поведению, если не заметить затенения, которое часто является нежелательным поведением и **местом возникновения трудноотслеживаемых багов**.

**Пример:**
```go
package main

import "fmt"

func main() {
	counter := 100
	if true {
		counter := 200 // Затеняет внешнюю counter
		message := fmt.Sprintf("Внутри: counter = %d", counter)
		fmt.Println(message)
	}
	message := fmt.Sprintf("Снаружи: counter = %d", counter)
	fmt.Println(message)
}
```
**Вывод:**
```makefile
Внутри: counter = 200
Снаружи: counter = 100
```

**Комментарий:** Внутренняя `counter` (200) затеняет внешнюю (100), но внешняя сохраняет своё значение после выхода из блока. Затенение может запутать, если не следить за областями видимости.

#### Блоки и их влияние на область видимости

Блок в Go — это код, заключённый в фигурные скобки `{}`, например, в конструкциях `if`, `for`, `switch` или просто в отдельном блоке. Блоки создают новую область видимости. Переменные, объявленные внутри блока, существуют только в пределах этого блока и недоступны снаружи. Это помогает изолировать переменные, избегая конфликтов имён и повышая безопасность кода. Внешние переменные, объявленные до блока, остаются доступными внутри него, но изменения внутри блока могут затенять их. Блоки могут быть вложенными, что создаёт иерархию областей видимости, где внутренняя область видит внешнюю, но не наоборот.

**Пример 1:**
```go
package main

import "fmt"

func main() {
	outerVar := 5
	if outerVar > 0 {
		innerVar := 10 // Доступна только внутри блока if
		message := fmt.Sprintf("Внутри: outerVar = %d, innerVar = %d", outerVar, innerVar)
		fmt.Println(message)
	}
	message := fmt.Sprintf("Снаружи: outerVar = %d", outerVar)
	fmt.Println(message)
	// fmt.Println(innerVar) // Ошибка: innerVar не определена
}
```
**Вывод:**
```makefile
Внутри: outerVar = 5, innerVar = 10
Снаружи: outerVar = 5
```

**Комментарий:** `innerVar` существует только внутри блока `if`. Попытка использовать её снаружи вызовет ошибку компиляции. Внешняя `outerVar` доступна внутри блока, что показывает, как внешние переменные видимы во вложенных областях.

**Пример 2: Сложный пример с вложенными блоками**

```go
package main

import "fmt"

func main() {
	message := "Начало"
	{
		message := "Внешний блок"
		for i := 1; i <= 1; i++ {
			if true {
				message := "Внутренний блок if"
				{
					message := "Самый внутренний блок"
					fmt.Println("Самый внутренний блок: message =", message)
				}
				fmt.Println("Внутренний блок if: message =", message)
			}
			fmt.Println("Цикл: message =", message)
		}
		fmt.Println("Внешний блок: message =", message)
	}
	fmt.Println("Вне блоков: message =", message)
}
```
**Вывод:**
```cmake
Самый внутренний блок: message = Самый внутренний блок , tempVar = 102
Внутренний блок 2: message = Внутренний блок if , innerVar = 2
Внутренний блок 1: message = Внешний блок , sum = 1
Внешний блок: message = Внешний блок , sum = 1
Вне блоков: message = Начало , outerVar = 100
```

**Комментарий:** Пример демонстрирует вложенные блоки: внешний блок, цикл `for`, блок `if` и ещё один вложенный блок. Переменная `message` затеняется в каждом блоке, создавая разные значения. В каждом блоке переменная выводится и наглядно видно то, что она принимает различные  значения, не изменяя значения в других блоках.
