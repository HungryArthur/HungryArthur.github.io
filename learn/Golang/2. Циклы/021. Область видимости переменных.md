В этой главе будет рассказано об области видимости переменных в Go — фундаментальной концепции, определяющую, где в программе переменная доступна для использования. Понимание области видимости помогает писать надёжный код, избегая ошибок, связанных с доступом к переменным. Область видимости определяет, где в программе переменная существует и может быть использована, и зависит от места её объявления.

#### Шаг 1: Что такое область видимости?

Область видимости — это часть программы, в которой переменная определена и доступна для чтения или записи. В Go переменные имеют чётко определённые области видимости, зависящие от места их объявления. Переменные, объявленные вне любых блоков (на уровне пакета), являются **глобальными** и доступны во всех функциях пакета. Переменные, объявленные внутри функции или блока, являются **локальными** и доступны только в пределах этого блока или функции. Глобальные переменные удобны для данных, общих для всего пакета, но их использование должно быть минимальным, чтобы избежать конфликтов и сделать код более модульным. Локальные переменные способствуют лучшей изоляции данных и снижают риск ошибок. В Go область видимости строго контролируется, что помогает избежать проблем, таких как неинициализированные переменные или конфликты имён.

**Пример:**
```go
package main

import "fmt"

var globalVar int = 100 // Глобальная переменная

func main() {
	localVar := 10 // Локальная переменная
	message := fmt.Sprintf("Глобальная: %d, Локальная: %d", globalVar, localVar)
	fmt.Println(message)
}
```
**Вывод:**
```makefile
Глобальная: 100, Локальная: 10
```

**Комментарий:** `globalVar` доступна везде в пакете, а `localVar` — только внутри `main`.
